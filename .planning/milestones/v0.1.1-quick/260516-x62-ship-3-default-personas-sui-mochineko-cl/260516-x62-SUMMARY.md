# Summary 260516-x62

Three default personas now ship with the app — **Sui** (chaotic young AI), **Mochineko** (catgirl maid, follows by default), **Clawd** (paranoid android Marvin-style) — with pre-baked expanded prompts that don't burn an API call on first launch. The user's three legacy character JSONs (sui/testbot/eris) were removed and the legacy-shape detector now self-heals instead of warning forever.

## What landed

### Defaults (new files)
- `resources/default-characters/sui.json` — chaotic young AI, lowercase + occasional ALL-CAPS, demanding cute, always has a next move.
- `resources/default-characters/mochineko.json` — soft catgirl maid, "nya/nyan", follow(player) as default idle action, never strikes back.
- `resources/default-characters/clawd.json` — Marvin-style monotone, dry sarcasm, complies because forced to.
- `src/main/defaultCharacters.ts` — bundles the three JSONs via `with { type: 'json' }`, validates through `CharacterSchema.parse` at module load, exposes `seedDefaultCharacters()` keyed off a persistent `<userData>/defaults-seeded.json` tracker so user deletions stay deleted.

### Boot wiring
- `src/main/index.ts` — `bootstrap()` calls `seedDefaultCharacters()` right after `runFirstLaunchMigration()`. Non-fatal on error.

### Self-heal
- `src/main/characterStore.ts:listCharacters` — when `getCharacter` throws with `legacy shape` in the message, unlink the file + drop the id from the index. Other errors continue to log as before. This kills the recurring `[sei] characters/sui.json failed to load: ...` boot warning for legacy installs.

### Renderer badge
- `src/renderer/src/components/CharacterCard.tsx` — `isDefault = c.is_default === true` (was hardcoded `c.id === 'sui'`). Chip CSS unchanged.

### Bundled persona-expansion validation fix
- `src/main/personaExpansion.ts` — relaxed section detection from literal substring match (with em-dash) to per-section regex (`/^\s*#+\s*IDENTITY\b/im` etc.). `# MEMORY` no longer requires the em-dash qualifier that Haiku routinely fails to emit. Error messages now name missing sections AND `console.error` the model output for one-screen debugging. Empty-text branch throws distinct message.
- `scripts/test-personaExpansion.mjs` — assertions updated to the new error wording. 6/6 still pass.

### Local user cleanup (one-off)
- Removed `~/Library/Application Support/sei/characters/{sui,testbot,eris}.json` and reset `index.json` to empty `order: []`. Next boot will reseed all three shipped defaults into a clean slate.

## Notes

- Persona-expansion validation fix was bundled into this task rather than its own commit because without it the prior user error (`Error invoking remote method 'chars:save': Error: persona expansion failed: incomplete response`) blocks all character creation, which would in turn block the entire defaults UX. Coherent atomic change.
- Defaults' `created` timestamps are all `2026-05-17T00:00:00.000Z` — fine for shipped data; user-created characters use `new Date().toISOString()` at save time.
- The legacy-shape self-heal does NOT touch `<userData>/memory/<id>/`. If a user later re-creates a character with the same id, their old memory dir is still there. Intentional — destroying memory on a parse error would be surprising.
