<objective>
Two related issues observed in a live session:

1. **Bot doesn't notice events between snapshots.** Haiku gets a fresh full snapshot every tick but doesn't reliably diff two snapshots to infer "wool went 1→2, sheep gone = killed" or "received netherite_sword". Fix: explicit `recent_events:` line in each snapshot derived from the previous snapshot of the same orchestrator instance.

2. **Bot drops the prior task after a chat interrupt.** Sequence: "kill sheep" → started → "give me sword first" → drops sword → asks "what next?" instead of resuming. The interrupt-and-resume code IS landed (b0c52f3) and the loop messages array is preserved across the abort, so the model CAN see prior tool_uses. The miss is salience: the prompt says "you may resume" but the prior task is buried in tool_use history. Fix: when synthesizing the `PLAYER INTERRUPT:` user turn, include an explicit `prior_task:` line summarizing the most recent in-flight intent so the model has a single-line cue to decide on.

Purpose: Two small, surgical changes — one observer (deltas), one prompt-shape (prior_task hint). Neither redesigns the FSM or orchestrator; both are non-interruptive (no new dispatch events).

Output:
- `recent_events:` line in every snapshot showing kills/inventory deltas/hp damage since the previous snapshot.
- `prior_task:` line in the synthesized PLAYER INTERRUPT user turn whenever an interrupt occurs mid-task with a known in-flight intent.
</objective>

Add a new exported factory `createSnapshotComposer({ bot })` that wraps the existing logic with per-instance previous-state memory. The existing `composeSnapshot(bot, opts)` MUST remain exported and stateless (other callers / tests rely on it — leave its current behavior identical).

Internal state held by the composer (closure variables):
- `prevInventory: Record<string,number> | null`
- `prevHp: number | null`
- `prevEntityIds: Set<number> | null`         // ids of mobs (non-player) that were within follow/attack-relevant radius
- `prevEntityMeta: Map<number, { name: string }>` // remember mob names so we can label disappearances after the entity is gone from bot.entities

Expose two methods:
- `next(opts)`: composes the next snapshot string, computes deltas vs previous state, injects a `recent_events:` line (only if any delta exists), then updates internal state. Returns the snapshot string. Same `opts` shape as composeSnapshot.
- `reset()`: clears internal state (used on death / disconnect — but wiring `reset()` is optional in this plan; just expose it).

Delta computation rules (deterministic, no LLM, no events fired):

1. **Inventory deltas (gains and losses):**
   - For each key in `currInventory`: gain = curr - (prev[k] ?? 0); if gain > 0, emit `+${gain} ${key}`.
   - For each key in `prevInventory` not in curr (or with curr < prev): loss = prev - (curr ?? 0); if loss > 0, emit `-${loss} ${key}`.
   - Cap to first 6 entries to keep the line tight; if more, append `(+N more)`.

2. **Kills (entity disappearance heuristic):**
   - A mob "disappeared" if its id was in `prevEntityIds` and is NOT in `currEntityIds` AND `bot.entities[id]` is undefined or has `isValid === false`.
   - Use `prevEntityMeta` to recover the name for the printed line (since the entity object is gone).
   - **Players never count as kills.** Skip any entity whose prev-meta `name === undefined` while it had a `username` — track only mobs (entries where `entity.username` was falsy when sampled).
   - Emit as `killed ${name}` for each disappearance. If multiple of same name, group: `killed sheep ×2`.
   - Caveat: this is a heuristic — a sheep that simply walked out of 24-block radius will also "disappear." That's fine for v1: the prompt change in Task 2 will say "recent_events is observational, not authoritative." Do NOT try to confirm via death event in this plan; that requires bot.on('entityDead') wiring (deferred — see Limitations note below).

3. **HP loss:**
   - If `prevHp != null && currHp < prevHp`: emit `hp -${prevHp - currHp}`.
   - HP gains (regen) are noisy; do NOT emit.

Render rules:
- If no deltas at all, do NOT emit the `recent_events:` line (keep snapshot tight when nothing happened).
- If any deltas: insert `recent_events: ${join(events, '; ')}` immediately after the `last_action_result` line (or at the end if no last_action_result). Do NOT put it before vitals/inventory — the LLM should see the current state first, then the deltas as commentary.

State update rules (do this AFTER composing the line, not before):
- Update `prevInventory` to curr.
- Update `prevHp` to curr.
- Rebuild `prevEntityIds` and `prevEntityMeta` from the FULL `Object.values(bot.entities)` list filtered to non-self, non-player, in-range (use radius: 24 to match the existing snapshot), so disappearances detect even mobs that were nearby last tick but not in the top-6 displayed.
- Players ARE excluded from kill tracking but should still NOT contaminate the "mob" id set — record `entity.username` falsy as the gate.

Refactor `composeSnapshot(bot, opts)` to call `createSnapshotComposer({ bot })` once and run `.next(opts)` on a temporary composer? **NO** — keep `composeSnapshot` as a plain stateless function (its current body, unchanged). Implement `createSnapshotComposer` as a separate factory that internally calls `composeSnapshot` to build the base string, then post-processes by inserting the `recent_events:` line. This avoids regressing every existing call site and any verify/test scaffold that calls composeSnapshot directly.

Concrete shape:
```js
export function createSnapshotComposer({ bot }) {
  let prevInventory = null
  let prevHp = null
  let prevEntityIds = null
  let prevEntityMeta = new Map()

  function sampleMobs() {
    const me = bot.entity
    if (!me) return { ids: new Set(), meta: new Map() }
    const ids = new Set()
    const meta = new Map()
    for (const e of Object.values(bot.entities ?? {})) {
      if (!e || e === me || !e.position) continue
      if (e.username) continue   // players excluded
      if (e.position.distanceTo(me.position) > 24) continue
      ids.add(e.id)
      meta.set(e.id, { name: e.name ?? `entity-${e.id}` })
    }
    return { ids, meta }
  }

  function computeEvents(currInv, currHp, currMobs) { /* …rules above… */ }

  return {
    next(opts = {}) {
      const base = composeSnapshot(bot, opts)
      const currInv = inventory(bot)
      const currHp = Math.round(bot.health ?? 0)
      const currMobs = sampleMobs()
      const events = computeEvents(currInv, currHp, currMobs)

      // Update state AFTER computing.
      prevInventory = currInv
      prevHp = currHp
      prevEntityIds = currMobs.ids
      prevEntityMeta = currMobs.meta

      if (events.length === 0) return base
      const line = `recent_events: ${events.join('; ')}`
      // Insert after last_action_result if present, else append.
      const lines = base.split('\n')
      const idx = lines.findIndex(l => l.startsWith('last_action_result:'))
      if (idx >= 0) lines.splice(idx + 1, 0, line)
      else lines.push(line)
      return lines.join('\n')
    },
    reset() {
      prevInventory = null; prevHp = null
      prevEntityIds = null; prevEntityMeta = new Map()
    },
  }
}
```

   Concrete shape:
   ```js
   function extractPriorTask(loop) {
     const msgs = loop._internal.messages
     for (let i = msgs.length - 1; i >= 0; i--) {
       const m = msgs[i]
       if (m.role !== 'assistant' || !Array.isArray(m.content)) continue
       for (let j = m.content.length - 1; j >= 0; j--) {
         const blk = m.content[j]
         if (!blk || blk.type !== 'tool_use') continue
         if (blk.name === 'say' || blk.name === 'setGoals') continue
         if (blk.name === 'handOffToMovement') {
           const intent = String(blk.input?.intent ?? '').slice(0, 120)
           return intent ? `intent="${intent}"` : null
         }
         // Combined-mode movement action
         const a = blk.input ?? {}
         const parts = []
         if (typeof a.entity === 'string') parts.push(a.entity)
         else if (typeof a.target === 'string') parts.push(a.target)
         else if (typeof a.block === 'string') parts.push(a.block)
         else if (typeof a.player === 'string') parts.push(a.player)
         if (typeof a.times === 'number') parts.push(`times=${a.times}`)
         return `${blk.name}${parts.length ? ' ' + parts.join(' ') : ''}`.slice(0, 120)
       }
     }
     return null
   }
   ```
