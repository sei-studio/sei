# Phase 9 Verification — Goal-Backward Audit

> **Status: `human_needed`**
>
> Every code-side gate has automated proof (master `npm run verify:phase9`
> chains all 4 sub-harnesses + typecheck passes both projects); but the
> ultimate ROADMAP success criterion — "the bot joins your LAN world wearing
> the chosen skin under the chosen username, visible to the host" — requires
> a live Minecraft client to confirm. The DEFER-TO-LIVE list at the bottom
> of this file is the bounded set of in-game smoke tests a developer or
> end-user must run before declaring Phase 9 fully shipped.

---

## Phase 9 Goal (verbatim from ROADMAP)

> Bot's custom skin and username are visible to the host in their own LAN
> world, with zero manual config beyond clicking through Sei's first-launch
> wizard. User workflow: download Sei → setup wizard auto-installs Fabric
> Loader + CustomSkinLoader into their MC profile (mac & windows) → upload
> or search a skin in Sei's character page → preview in 3D → on next MC
> launch select the new "Sei (Fabric Loader)" profile → bot joins their
> world wearing the chosen skin under any chosen username. Works on
> vanilla MC and CurseForge instances (Pixelmon etc.).

## DEFER-TO-LIVE Summary

These tests cannot be automated — they require a developer's local environment
with a Minecraft launcher installed (and at one point, a real LAN session
visible from the host's MC client). Numbered in the order a developer should
run them:

### 1. Wizard auto-open on first launch (macOS)

- Run `npm run dev` on macOS with the Vanilla Launcher installed and at
  least one MC profile that's been launched once (so the bundled JRE
  exists).
- Expected: the SetupWizardModal opens automatically (`hasApiKey === true`
  AND `!hasRunOnce` AND `installs.length > 0`).
- Verify: wizard scans your installs; the vanilla launcher install appears
  in the "Pick installs" step.

### 2. Live Fabric + CustomSkinLoader install (BLOCKER 3 + Plan 04 live gates)

- From the wizard, click "Continue" on the pre-selected vanilla install.
- Expected: Fabric installer downloads from `meta.fabricmc.net`, runs
  using the bundled JRE found by `findBundledJava` (BLOCKER 3 — should
  NOT require system-PATH `java`), CSL JAR appears in
  `~/Library/Application Support/minecraft/mods/`, CSL config appears at
  `~/Library/Application Support/minecraft/config/CustomSkinLoader/CustomSkinLoader.json`.
- Verify the config contains `"root": "http://127.0.0.1:<port>"` AND
  `"type": "Legacy"` (WARNING 6 — `Legacy`, NOT `CustomSkinAPI`).

### 3. Launch MC + select Fabric profile

- Open the Mojang Launcher → select the newly-installed `fabric-loader-...`
  profile → click Play. Verify Minecraft starts with Fabric Loader.

### 4. Open to LAN

- In Minecraft, open a single-player world, click Open to LAN. Verify the
  port is broadcast (Sei's LanModal pill should flip to "connected").

### 5. Summon a Sei character with a custom Mojang skin (WARNING 8 live gate)

- In Sei, open a character page (e.g. Sui).
- In the Skin & Username editor, type `Notch` into the Mojang-username
  search field → click "Look up" → 3D preview shows Notch's current skin.
  (Notch's account is one of the oldest Mojang accounts; if the skin is
  served as legacy 64×32, the normalize64x64 path exercises WARNING 8 in
  the real Mojang pipeline.)
- Click "Apply skin". Toast: "Skin applied. It'll show up on the next bot
  summon."
- Click Summon. Bot joins the LAN world.

### 6. Verify the bot wears the chosen skin under the chosen username

- In Minecraft, look at the bot. Verify:
  - The username above the bot's head is the persona's `username` (e.g.
    `Sui`), not a sanitized fallback.
  - The bot's skin is Notch's skin (or whatever you applied), NOT a
    default Minecraft skin.
- This is the ROADMAP success criterion (RM-W5 + RM-W6 above).

### 7. Cancel mid-install (BLOCKER 2 live gate)

- Re-run the wizard from Settings → "Re-run setup". The wizard re-detects
  installs and lets you opt in again.
- Click Continue. While the Fabric installer download is in flight, click
  Cancel in the wizard footer.
- Expected:
  - The install row flips to `Cancelled` (muted StatusPill).
  - The main-process log shows the `java` child process being SIGTERM'd
    via `execFile`'s signal option.
  - Re-running the wizard works (no zombie state).

### 8. Port-drift recovery (WARNING 7 live gate)

- Quit Sei. Restart Sei. The OS will pick a fresh ephemeral port for the
  skin server.
- Expected: the main log emits the literal anchor
  `skin server port drift detected, rewriting N CSL configs`.
- Open the CSL config on disk; verify the `loadlist[0].skin` URL now
  reflects the new port.

### 9. Windows VM smoke (cross-platform live gate)

- Repeat tests 1–8 on a Windows VM with the Vanilla Launcher installed at
  `%APPDATA%\.minecraft`. Verify:
  - Path detection works under `%APPDATA%\.minecraft` and (if installed)
    `%USERPROFILE%\curseforge\minecraft\Instances\`.
  - Fabric installer runs via the bundled JRE under
    `%APPDATA%\.minecraft\runtime\java-runtime-<component>\windows-x64\...`.
  - CSL config path uses Windows separators (`config\CustomSkinLoader\CustomSkinLoader.json`).
  - The bot joins a Windows-hosted LAN world wearing the chosen skin.
- Deferred per Phase 8 Wave 3 — same Windows VM constraint.
