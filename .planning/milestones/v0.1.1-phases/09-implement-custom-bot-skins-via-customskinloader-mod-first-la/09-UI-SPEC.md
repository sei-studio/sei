# Phase 9 ‚Äî UI Design Contract

> Visual and interaction contract for the CustomSkinLoader setup wizard, the per-persona Skin & Username editor (with 3D preview), and MC-install status indicators. Generated by gsd-ui-researcher; consumed by gsd-planner, gsd-executor, and gsd-ui-checker.

This phase **extends** Sei's existing Electron renderer (Phase 4 stack: React + CSS Modules + custom CSS-variable tokens). No new design system is introduced. All new surfaces inherit the tokens in `src/renderer/src/styles/tokens.css` and the global resets in `global.css` (sharp edges D-28, body wallpaper gradient, custom scrollbars, light/dark theme toggle).

## Design System

| Property | Value |
|----------|-------|
| Tool | none (existing custom CSS-tokens system from Phase 4 04-UI-SPEC) |
| Preset | not applicable ‚Äî shadcn was NOT initialized; Sei ships its own primitive set under `src/renderer/src/components/` |
| Component library | none (custom primitives: `Button`, `TextField`, `QuestionShell`, `StepDots`, scrim+modal pattern from `LanModal`/`EditCharacterModal`/`DeleteConfirmModal`) |
| Icon library | inline SVG components in `src/renderer/src/components/icons.tsx` (e.g. `SparkleIcon`, `BackIcon`, `ArrowIcon`); **new icons for Phase 9 added inline to that same file** ‚Äî no external icon package |

New components live alongside existing ones in `src/renderer/src/components/` with paired `.module.css` files.

| Token | CSS var | Value | Usage in Phase 9 |
|-------|---------|-------|------------------|

| sm | `--space-sm` | 8px | Compact element spacing ‚Äî gap between status pill dot and pill label, inline `<kbd>`-like tags |
| md | `--space-md` | 12px | Default text-to-action vertical rhythm inside wizard step body |

| xl | `--space-xl` | 32px | Major vertical break between the Skin editor and the rest of the CharacterPage; padding inside the wizard "Step complete" panel |

**Wizard modal sizing:** the wizard is a centered modal-style flow. Modal max-width: `680px` (wider than `QuestionShell`'s 520px so multi-row install detection lists fit). Min-height: `520px` to prevent layout jump between steps that have very different content heights.

Sized in design tokens as `--skin-preview-w: 240px; --skin-preview-h: 320px;` ‚Äî declared **once** at the top of the `SkinPreview.module.css` file as locally-scoped custom properties; NOT promoted to `tokens.css` because they have a single call-site.

Exceptions: none. Every Phase 9 spacing value resolves to one of the eight tokens above or to a Phase-local custom property documented in component CSS.

### Accent reserved-for list (explicit)

`var(--accent)` is used in Phase 9 **only** for:

1. The wizard's primary forward CTA on the final step (`Button kind="accent"`: "Finish setup")
2. The Skin editor's "Apply skin" primary CTA on the persona page (`Button kind="accent"`)
3. The pixel-font wordmark on the wizard "Welcome" panel (`SeiPixelMark` reused)
4. The focused state of text inputs (border + 1px `--accent-soft` glow) ‚Äî Phase 4 behavior, inherited
5. The "currently-selected MC install" row's left-edge accent bar (2px wide)

Accent is **NOT** used for:
- Step indicators (use `--text-2` and `--accent` mix in `StepDots` ‚Äî that component already encodes the rule)
- Status dots (each uses its semantic color: green/red/warn/muted)
- Hover states of non-CTA buttons (use `--rail-active` background instead)
- Selection highlights inside text inputs (browser default)
- Links or text emphasis

### Destructive color usage

`var(--red)` is reserved for:
- "Remove skin" button inside the Skin editor (matches the `EditCharacterModal` Danger section pattern)
- Failed mod-download error banner
- Failed Mojang username lookup ("No Minecraft account named X")
- Status dot when a previously-enabled MC install is now in a broken state (mod missing or version mismatch)

`var(--red)` is **never** used for:
- "Cancel" or "Back" buttons (use `kind="quiet"`)
- Neutral validation hints (use `--text-2`)

### Status dot system (D-22 family, extended)

All status indicators in Phase 9 use the 8px-dot + label pill pattern from `LanModal` / `CharacterPage` model row:

| Dot color | Label | Meaning |
|-----------|-------|---------|
| `var(--green)` | "Sei enabled" / "Setup complete" / "Mod installed" | Detected install + CustomSkinLoader present + config correct |
| `var(--warn)` | "Version drift" / "Re-install recommended" | Install detected but mod is older than bundled version, OR Fabric Loader version doesn't match recommended |
| `var(--red)` | "Setup failed" / "Mod missing" | Install detected but mod absent (e.g. user manually deleted), OR last setup attempt errored |
| `var(--muted)` | "Not enabled" / "Detected, not configured" | Install detected; user hasn't opted to enable Sei here |
| `var(--text-2)` (slow pulse) | "Installing‚Ä¶" / "Detecting‚Ä¶" / "Searching skin‚Ä¶" | In-flight async work |

## Layout Contracts

### Setup wizard modal

- **Trigger:** automatically opens on first launch when `userData/skin-setup-state.json` is absent. Manual re-trigger from Settings ‚Üí "Set up Minecraft skins" row.
- **Dismissal:** ESC closes the wizard (matches `LanModal`/`EditCharacterModal`). Closing mid-wizard preserves "user dismissed" so next launch does not auto-reopen ‚Äî user must re-run from Settings.
- **Modal dimensions:** `max-width: 680px`, `min-height: 520px`, centered. Padding `24px 40px 40px` (matches `CharacterPage`).
- **Scrim:** `rgba(0,0,0,0.45)` over the existing desktop wallpaper. Same scrim opacity as `LanModal`.
- **Step transitions:** crossfade `200ms ease` between steps. No horizontal slide (avoid motion sickness; Sei's existing transitions are all opacity-based).
- **Step indicator:** `STEP {n} / 5` in pixel font, top-left of step body. Total step count is fixed at 5 (Welcome, Detecting, Pick installs, Installing, Done). Branch states (1b, 3b) do not advance the counter.

## Accessibility Contracts

- **Keyboard navigation:** the entire wizard MUST be traversable with Tab/Shift+Tab and Enter/Space. ESC exits at any step (same dismissal semantics as ESC on `LanModal`).
- **Focus trap:** active inside `SetupWizardModal` while open ‚Äî first focusable on open is the primary CTA. Re-focus the trigger element on close (Settings row or Settings ‚Üí "Re-run setup" button).
- **Live region:** during step 1 (Detecting) and step 3 (Installing), wrap the in-flight status text in `role="status" aria-live="polite"` so screen readers narrate progress.
- **3D preview:** `<canvas>` has `role="img" aria-label="3D preview of {persona_name}'s skin"`. The 2D fallback `<img>` has the same alt text. The 3D canvas itself is NOT keyboard-focusable in v1 (orbit interaction is mouse/touch only); the persona name + skin source state provide sufficient context for screen readers.
- **Upload zone:** has `role="button"` AND a hidden `<input type="file" accept="image/png">` for keyboard activation. `aria-describedby` points at the secondary "Files stay on your computer" hint.
- **Status pills:** dot color carries semantic meaning but ALL pills also carry the textual label ‚Äî no information is conveyed by color alone. Verify with `prefers-contrast: more` smoke test that status pills remain legible.
- **Color contrast:** all text-over-background combinations specified above are inherited from Phase 4 tokens, which passed WCAG AA on the existing UI. New surfaces (install rows, upload zone) MUST be re-verified by the checker against AA at the body-text size.
