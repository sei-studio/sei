    ```ts

    const sessions = new Map<string, AbortController>();

    /** Register a new session and return its AbortController. Caller stores the sessionId. */
    export function registerWizardSession(sessionId: string): AbortController { /* ... */ }

    /** Abort a session by id. Returns true if the session existed and was aborted; false if no such session. */
    export function abortWizardSession(sessionId: string): boolean { /* ... */ }

    /** The orchestrator. */
    export async function runWizardInstall(args: {
      sessionId: string;
      installIds: string[];
      skinServerBaseUrl: string;
      onProgress: (ev: WizardProgressEvent) => void;
    }): Promise<{ results: WizardInstallResult[] }>;
    ```

    Implementation outline:

    1. `registerWizardSession(sessionId)`:
       - If a session with that id already exists, abort the previous one (defensive — should never happen with crypto.randomUUID()).
       - Create a new `AbortController`. Store in the map keyed by sessionId.
       - Return the controller.

    2. `abortWizardSession(sessionId)`:
       - `const ctl = sessions.get(sessionId)`. If undefined → return false.
       - `ctl.abort(new Error('WIZARD_CANCELLED: user cancelled'))`. Sessions self-clean inside `runWizardInstall`'s finally block, so we don't delete here.
       - Return true.

    3. `runWizardInstall(args)`:
       - Validate `args.sessionId` non-empty; if empty, throw `WIZARD_PERMISSION_DENIED: invalid session`.
       - Call `registerWizardSession(args.sessionId)` to get the controller; pass `ctl.signal` to every Plan 04 call.
       - Call `scanMcInstalls()` to get the current install list. Filter to those whose `id` is in `args.installIds`. For any id in `args.installIds` that ISN'T in the scan result → emit `WizardProgressEvent { installId, stage: 'failed', error: 'MC_INSTALL_NOT_FOUND', message: 'Install no longer detected. Re-run setup.' }` and push the matching `WizardInstallResult` with ok=false; skip to next.
       - For each matched install, in series (NOT parallel — multiple `java -jar` runs concurrently could thrash and the user-facing UI shows one progress row at a time per UI-SPEC §InstallProgressList):
         * Pre-step: check `ctl.signal.aborted`. If aborted → emit `{ installId, stage: 'cancelled' }` and push `{ ok: false, error: 'FABRIC_INSTALL_FAILED', message: 'Cancelled.' }` to results. Skip remaining installs (break).
         * Emit `{ installId, stage: 'queued' }`.
         * Determine mc version: `install.mc_version ?? '1.21.4'` (fallback to a known-good version when launcher_profiles is unreadable). If still null, fail with `MC_INSTALL_NOT_FOUND` ("Couldn't determine MC version").
         * Determine loader kind:
           - vanilla install with `loader === null`: install Fabric first. Emit `{ stage: 'fabric-downloading', pct: 0 }`. Call `installFabricLoader({ mcInstall: install, mcVersion, signal: ctl.signal, onProgress: (pct) => onProgress({ installId, stage: 'fabric-downloading', pct }) })`. Catch and re-throw with stage `'failed'` + error `FABRIC_INSTALL_FAILED`. After success emit `{ stage: 'fabric-installing' }` then continue. Use 'fabric' as the loader kind for the next step.
           - vanilla install with `loader === 'fabric'`: skip Fabric step. Use 'fabric' as the loader kind.
           - curseforge install: use `install.loader` (could be 'forge' or 'fabric') as the loader kind. If `install.loader === null`, treat as 'forge' (CurseForge defaults to Forge for the vast majority of modpacks).
         * Emit `{ stage: 'mod-downloading', pct: 0 }`. Call `downloadCustomSkinLoader({ loaderKind, mcVersion, modsDir: path.join(install.path, 'mods'), signal: ctl.signal, onProgress: (pct) => onProgress({ installId, stage: 'mod-downloading', pct }) })`. Catch → fail with `MOD_DOWNLOAD_FAILED`.
         * Emit `{ stage: 'mod-placing' }` (the actual placement happened inside `downloadCustomSkinLoader` via fs.rename — this stage is purely a UI signal).
         * Emit `{ stage: 'config-writing' }`. Call `writeCustomSkinLoaderConfig({ mcInstallDir: install.path, loaderKind, skinServerBaseUrl: args.skinServerBaseUrl })`. Catch → fail with `WIZARD_PERMISSION_DENIED` (EACCES is the typical cause).
         * Emit `{ stage: 'done' }`. Push `{ installId, ok: true, installedFabricVersion?: ..., installedCslVersion: ... }` to results.
         * On AbortSignal mid-step (a downstream Plan 04 module rejects with a cancellation error): emit `{ stage: 'cancelled' }`, push `{ ok: false, error: 'FABRIC_INSTALL_FAILED' or 'MOD_DOWNLOAD_FAILED', message: 'Cancelled.' }`, and break the loop (don't continue to remaining installs — the user cancelled the whole run, not just this row).

       - After all installs processed: load current `WizardState`, update `hasRunOnce: true`, merge successful `installIds` into `enabledInstallIds` (dedupe), set `lastRunAt: new Date().toISOString()`, set `lastSkinServerPort: parseInt(new URL(args.skinServerBaseUrl).port, 10)`. Save via `saveWizardState`.
       - Finally block: `sessions.delete(args.sessionId)` to free the controller reference.
       - Return `{ results }`.

    - `wizard:cancel` IPC handler (Task 2) simply calls `abortWizardSession(sessionId)` and returns immediately. The in-flight `runWizardInstall` then rejects through Plan 04's signal-aware modules.

    ```ts
    // 1d. Port-drift detection (WARNING 7).
    // Across launches, the OS may give us a different port for the skin server (we bind port: 0).
    // If the wizard has run before AND the port changed, CSL configs are stale → rewrite them.
    if (skinServer) {
      try {
        const wizardState = await loadWizardState();
        if (wizardState.hasRunOnce && wizardState.lastSkinServerPort !== null && wizardState.lastSkinServerPort !== skinServer.port) {
          logger.info(`skin server port drift detected, rewriting ${wizardState.enabledInstallIds.length} CSL configs`);
          const { writeCustomSkinLoaderConfig } = await import('./customSkinLoader');
          const { scanMcInstalls } = await import('./mcInstallScan');
          const installs = await scanMcInstalls();
          const byId = new Map(installs.map(i => [i.id, i]));
          for (const installId of wizardState.enabledInstallIds) {
            const inst = byId.get(installId);
            if (!inst) {
              logger.warn(`port drift rewrite: install ${installId} no longer detected; skipping`);
              continue;
            }
            // Determine loaderKind: for vanilla, fabric. For CF, use whatever the scan detected.
            const loaderKind: 'fabric' | 'forge' = inst.kind === 'vanilla' ? 'fabric' : (inst.loader === 'fabric' ? 'fabric' : 'forge');
            try {
              await writeCustomSkinLoaderConfig({ mcInstallDir: inst.path, loaderKind, skinServerBaseUrl: skinServer.baseUrl });
              logger.info(`port drift rewrite: ${installId} → ${skinServer.baseUrl}`);
            } catch (err) {
              logger.warn(`port drift rewrite: ${installId} failed: ${(err as Error).message}`);
            }
          }
          // Update persisted state so next launch doesn't re-rewrite unless the port changes again.
          await saveWizardState({ ...wizardState, lastSkinServerPort: skinServer.port });
        }
      } catch (err) {
        logger.warn(`port drift detection failed: ${(err as Error).message}`);
        // Non-fatal — bot launches, just the configs may be stale. User can re-run wizard from Settings.
      }
    }
    ```
