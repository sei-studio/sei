# Phase 2.1 Plan 01: Observers & Cached-Prefix Substrate — Summary

**One-liner:** Built the pure-function observer layer (`vitals`/`inventory`/`world`/`blocks`/`entities`), the snapshot composer with monotonic `#N` handle table, the targeting resolver (`resolveBlock`/`resolveEntity`/`isStaleHandle`), and extended the cached system prefix with capability paragraph, Minecraft primer, and "still learning" persona trait — all the contracts Plans 02 and 03 will consume.

## What Was Built

### Observers (`src/observers/`)
- **`vitals.js`** — `vitals(bot)` returning `{ hp, food, xp:{level}, sleeping }`, hp/food rounded.
- **`inventory.js`** — `heldItem(bot)` (item + durability) and `inventory(bot)` (stack-summarized `{name: count}` map).
- **`world.js`** — `world(bot)` returning `{ pos, biome, time:{isDay,timeOfDay} }` with rounded coords and `'unknown'` biome fallback.
- **`blocks.js`** — `nearbyBlocks(bot, {radius=16,count=8,interesting})` using `bot.findBlocks` for the rendered slice + a higher-cap probe call to compute `+K more`. Exports `INTERESTING_BLOCK_NAMES` (logs/planks/ores incl. deepslate/16-color beds/utility blocks). Resolves `matching` via `minecraft-data` ids when available, with function-form fallback.
- **`entities.js`** — `nearbyEntities(bot, {radius=24,count=6})` filtering `Object.values(bot.entities)` by distance, sorted closest-first, returning `{entries, more}`.
- **`targeting.js`** — Single in-memory `_handles` Map; `setHandles(entries)` does per-snapshot replace; `getHandles()` for tests; `HANDLE_TTL_MS = 10_000`.
  - `resolveBlock(args, bot)` — explicit `(x,y,z)` → `bot.blockAt`; `target:"#N"` → handle lookup with TTL/kind check → `bot.blockAt(h.pos)`; `block:string` → `mcData.blocksByName[name].id` then `bot.findBlock({ matching:[id], maxDistance:32 })` (fallback to function-form match).
  - `resolveEntity(args, bot)` — `entity_id` → `bot.entities[id]`; `target:"#N"` → handle; `entity:string` → `bot.nearestEntity` matching `name` or `username`.
  - `isStaleHandle(args)` — true iff `args.target` starts with `#` AND handle is missing/expired.

### Snapshot composer (`src/observers/snapshot.js`)
- `composeSnapshot(bot, {goals, lastActionResult})` produces line-oriented text in D-26 order: pos, biome+time, vitals, optional `status: sleeping`, holding (with durability), inventory, nearby blocks (8 cap, `#N name @x,y,z`), nearby entities (6 cap), owner_goals, self_goals, optional last_action_result.
- Monotonic `#N` numbering across blocks then entities; each handle stamped `expiresAt = Date.now()+10s`; calls `setHandles()` once at the end (only mutation visible to callers).

### Cached prefix (`src/llm/persona.js`, `src/llm/anthropicClient.js`)
- `capabilityParagraph()` — single NL paragraph (~717 chars), enumerates active tier-1 capabilities and includes the inline can't list with all five required gerunds: **crafting, riding, enchanting, brewing, redstone**.
- `minecraftPrimer()` — ~1377 chars covering wood-by-biome, hostile mobs, tool→block matrix, day/night basics.
- `stillLearningLine()` — verbatim D-32 sentence ("You're new to this world and still learning…").
- All three implemented as module-level constants → byte-stable for prompt-cache stability.
- `buildCachedSystem(systemInstructions, personaText, capabilityParagraph, primer, learningLine, tools)` now emits **5 blocks** in order: instructions, `personaText\nlearningLine`, capability, primer, tool list — with `cache_control: {type:'ephemeral'}` on the LAST (tool) block per D-18.

## How `must_haves` Were Satisfied

| Truth | Evidence |
|-------|----------|
| `composeSnapshot` line-oriented snapshot under ~500 tokens with required fields | `src/observers/snapshot.js` composes pos/biome/time/vitals/holding/inventory/blocks/entities/goals; hard caps 8 blocks + 6 entities. |
| Snapshot handle table `#N -> {kind, pos\|entityId, expiresAt}` ~10s TTL | `targeting.js` `HANDLE_TTL_MS=10_000`; snapshot.js stamps `expiresAt = Date.now()+HANDLE_TTL_MS`; `resolveBlock`/`resolveEntity` call `lookupHandle` which checks TTL. |
| `resolveBlock` supports name / `#N` / explicit coord; `resolveEntity` supports name / `#N` / `entity_id` | Implemented in `targeting.js` per spec. |
| Cached prefix accepts capability + primer + learning BEFORE the breakpoint | `anthropicClient.buildCachedSystem` emits 5 blocks with `cache_control` on the 5th (tool) block. |
| Persona module exports stable byte-stable strings | `capabilityParagraph`/`minecraftPrimer`/`stillLearningLine` return module-level `const` strings. |

## Key Decisions Made

- **mcData fast-path with function-form fallback** for `nearbyBlocks` and `resolveBlock` — keeps `minecraft-data` optional/version-tolerant.
- **Per-snapshot replace** for the handle table (D-25 alternative), implemented in `setHandles` by clearing then writing all entries — simple, deterministic, avoids partial-state bugs.
- **`isStaleHandle` semantics** treat any missing/expired handle as stale; kind-mismatch is checked at resolve time inside `resolveBlock`/`resolveEntity` (not knowable from `args` alone).
- **Capability "can't" list uses gerund forms** (crafting/riding/enchanting/brewing/redstone) so the prose contains the canonical tokens that downstream verifier prompts can grep for.
