  <files>src/behaviors/dig.js, src/behaviors/place.js, src/behaviors/equip.js, src/behaviors/attack.js, src/behaviors/consume.js, src/behaviors/lookAt.js</files>

Each file exports a single async function with signature `(args, bot, config) => Promise<string>` and uses the timeout+abort race pattern from `src/behaviors/pathfind.js` (per D-24 / ADR #5).

Universal pattern boilerplate (apply to every handler):
```js
const signal = config?.signal
if (signal?.aborted) return 'aborted'
// ... (resolve target if any; if null, return 'stale target' if isStaleHandle(args) else 'no target')
const timeoutMs = args.timeout_ms ?? config?.<NAME>_timeout_ms ?? DEFAULT_TIMEOUT_MS
const op   = bot.<doSomething>(...).then(() => `<success result>`).catch(() => `<failure result>`)
const tmo  = new Promise(r => setTimeout(() => { try { /* cleanup */ } catch {}; r('timeout') }, timeoutMs))
const abrt = new Promise(r => { if (!signal) return; signal.addEventListener('abort', () => { try { /* cleanup */ } catch {}; r('aborted') }, { once: true }) })
return Promise.race([op, tmo, abrt])
```

Per-handler specifics:

1. **dig.js** (default 8000 ms, ref 2.1-RESEARCH.md Pattern 1):
   - Resolve target with `resolveBlock(args, bot)`; if null + isStaleHandle → 'stale target', else 'no target'.
   - Pre-check `bot.canDigBlock(block)` → if false return `cannot dig ${block.name}`.
   - **Single-flight guard (Pitfall 2):** if `bot.targetDigBlock != null` return `'busy digging'`. Do NOT await another dig — that's the fatal re-entry path.
   - Cleanup branches (timeout AND abort) call `bot.stopDigging()` inside try/catch.
   - Success: `dug ${block.name}`. Catch branch: `cannot dig ${block.name}`.

2. **place.js** (default 4000 ms):
   - Args allow `{ block: string (item name to place), against: same target shape as resolveBlock, faceVector?: {x,y,z} }`. Use a closed Zod schema (registered in Task 3) — handler can assume validated input.
   - Resolve referenceBlock via `resolveBlock(args.against, bot)` (against is a sub-object). If null → 'no target' / 'stale target'.
   - Verify the named item is in `bot.inventory.items()` — if missing return `no ${args.block} in inventory`.
   - Equip the item to hand first via `await bot.equip(item, 'hand')` (catch -> `'could not equip'`).
   - Default faceVector to `new Vec3(0, 1, 0)` (top face) when unspecified.
   - Call `bot.placeBlock(referenceBlock, faceVector)`. Success: `placed ${args.block}`. Catch: `cannot place ${args.block}`.
   - No special cleanup on timeout (placeBlock is fast).

3. **equip.js** (default 2000 ms):
   - Args: `{ item: string, destination: 'hand' | 'off-hand' | 'head' | 'torso' | 'legs' | 'feet' }`.
   - Find item via `bot.inventory.findInventoryItem(mcData.itemsByName[item]?.id, null, false)` — if null return `no ${item}`.
   - Race `bot.equip(invItem, destination)` with timeout. Success: `equipped ${item}`. Catch: `cannot equip ${item}`.

4. **attack.js** (default 6000 ms; ref Pitfall 5):
   - Resolve target via `resolveEntity(args, bot)`. If null → 'stale target' or 'no target'.
   - **Refuse Player targets:** if `entity.type === 'player'` (or `entity.username` truthy) return `cannot attack player`. This honors REQUIREMENTS "Auto-PvP against players" out-of-scope.
   - Pre-check distance: `bot.entity.position.distanceTo(entity.position) <= 3.5` else `target out of reach`.
   - Single swing only: `bot.attack(entity)` (synchronous in mineflayer; wrap in Promise.resolve). Cooldown awareness is the personality LLM's job; this handler does ONE swing per call.
   - Result: `attacked ${entity.name ?? 'entity'}`.

5. **consume.js** (default 4000 ms; ref Pitfall 3):
   - No target resolution.
   - Optional `args.item: string` — if provided, equip it first via `bot.equip(invItem, 'hand')`; if not provided, assume already-held food.
   - **Pre-call:** `bot.clearControlStates()` to avoid eat-while-moving silent failure.
   - Race `bot.consume()` with timeout. Success: `ate ${heldItem.name ?? 'food'}`. Catch: `could not eat`.

6. **lookAt.js** (default 2000 ms):
   - Args: either `{ x, y, z }` coord OR `{ target: '#N' }` entity handle OR `{ entity: name }` OR `{}` (refresh-only no-op).
   - For coord: `bot.lookAt(new Vec3(x,y+1.6,z))` (eye level offset).
   - For entity: resolveEntity → `bot.lookAt(entity.position.offset(0, entity.height ?? 1.6, 0))`.
   - For empty args: no rotation, just resolve `'looked'`.
   - Race vs timeout. Success: `looked`. Catch: `cannot look`.

All files import `Vec3` from 'vec3' (named export) and minecraft-data via `import mcDataLib from 'minecraft-data'` only where needed (equip needs item id lookup; alternatively use `bot.inventory.items().find(i => i.name === args.item)` and skip mcData entirely — that is preferred since it avoids a redundant dep import).

Constants: export `DEFAULT_TIMEOUT_MS` from each file so registry config can read defaults.

NO logging from handlers (Phase 2 pattern: orchestrator/FSM handle logging; handlers stay pure).

4. **container.js** — three exported actions sharing module state, plus `closeContainerSession`:

```js
const SESSION = { container: null, blockPos: null }

export async function closeContainerSession() {
  if (SESSION.container) {
    try { await SESSION.container.close() } catch {}
    SESSION.container = null
    SESSION.blockPos = null
  }
}

export async function openContainerAction(args, bot, config) { /* ... */ }
export async function depositItemAction(args, bot, config)   { /* ... */ }
export async function withdrawItemAction(args, bot, config)  { /* ... */ }
```

- **openContainerAction** (default 6000 ms; ref Pitfall 1, Pitfall 6, Pattern 6):
  - Close any prior session via `closeContainerSession()` first (Pitfall 6 — never overlap).
  - Resolve target via `resolveBlock(args, bot)`. Null → stale/no.
  - Distance check: `bot.entity.position.distanceTo(target.position) <= 4` else `target out of reach`.
  - Race `bot.openContainer(target)` with timeout. On success: store `SESSION.container = container; SESSION.blockPos = target.position; return 'opened ' + target.name`.
  - On timeout/abort: call `closeContainerSession()` to drain a half-opened window. Return `timeout` / `aborted`.

- **depositItemAction** (default 4000 ms):
  - If `!SESSION.container` return `'no container open'`.
  - Args: `{ item, count=1 }`.
  - Find inventory item by name → if absent `'no ${item} to deposit'`.
  - Race `SESSION.container.deposit(itemId, null, Math.min(count, invItem.count))` with timeout.
  - Success: `deposited ${actualCount} ${item}`. Catch: `deposit failed`.
  - On timeout: do NOT close the session (the deposit may still complete server-side; let chain-end cleanup handle it).

- **withdrawItemAction** (default 4000 ms):
  - If `!SESSION.container` return `'no container open'`.
  - If `SESSION.container.containerItems().filter(i => i.name === args.item).length === 0` return `'chest empty'` (or `'no ${item} in chest'` — pick one wording and stay consistent; D-35 example uses `'chest empty'` for the empty case).
  - Race `SESSION.container.withdraw(itemId, null, count)` with timeout.
  - Success: `withdrew ${count} ${item}`. Catch: `withdraw failed`.
  - Inventory full case: catch and return `inventory full` if the prismarine error message includes "no room"; otherwise `withdraw failed`.

Honor `config.signal` in all four functions. The abort branch on container ops should also `closeContainerSession()`.

The orchestrator (Plan 03) will import `closeContainerSession` and call it on `chains.end(chainId)` and on `signal.aborted` paths to enforce Pitfall 6 invariant globally.

Use minecraft-data for item id lookup OR use `bot.inventory.findInventoryItem(itemId, null, false)` after getting id from `mcDataLib(bot.version).itemsByName[name]?.id`. Either is fine — pick one approach and stay consistent in this file.
