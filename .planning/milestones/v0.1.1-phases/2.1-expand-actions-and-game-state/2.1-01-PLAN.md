// args shape accepted by resolveBlock — every tier-1 block-target action accepts this
// at minimum (handlers may add their own fields like maxDistance defaults).

  args: { block?: string, target?: string /* "#N" */, x?: number, y?: number, z?: number, maxDistance?: number },
  bot: Bot
): Promise<Block | null>

export function resolveEntity(
  args: { entity?: string, target?: string /* "#N" */, entity_id?: number },
  bot: Bot

export function setHandles(
  entries: Array<[string, { kind: 'block', pos: Vec3, expiresAt: number }
                       | { kind: 'entity', entityId: number, expiresAt: number }]>
): void

- `targeting.js` is the load-bearing module. Implements an in-memory `_handles: Map<string, HandleEntry>`, with `setHandles(entries)` replacing the table and `getHandles()` for tests. Implements:

If `minecraft-data` import is awkward, use the function-form fallback: `bot.findBlock({ matching: (b) => b.name === args.block, maxDistance })`. Choose ONE approach and document the choice in a comment.
  - `resolveEntity(args, bot)` — `entity_id` -> `bot.entities[entity_id] ?? null`; `target` starts with `#` and kind 'entity' and not stale -> `bot.entities[h.entityId]`; `entity` string -> `bot.nearestEntity(e => (e.name ?? '').toLowerCase() === entity.toLowerCase())` (also try `e.username` for player names — but per CONTEXT/REQUIREMENTS, attackEntity must refuse Player targets; resolveEntity itself can return them and the handler refuses).
  - `isStaleHandle(args)` — returns true iff `args.target` is a `#`-prefixed string and either the handle is missing OR `expiresAt < Date.now()` OR kind mismatches. Used by handlers to choose result string `"stale target"` vs `"no target"`.
  - Constant `HANDLE_TTL_MS = 10_000`.

All observers are pure (no setHandles call). Only `composeSnapshot` (Task 2) calls `setHandles`.

Use Vec3 via `import { Vec3 } from 'vec3'` (named export). Use minecraft-data via `import mcDataLib from 'minecraft-data'`.

No external deps to install — all transitive via mineflayer.
