# Phase 2.1 Plan 02: Tier-1 Action Handlers — Summary

**One-liner:** Implemented the 12 mineflayer tier-1 action handlers (dig/place/equip/attackEntity/consumeItem/lookAt/dropItem/activateItem/sleep + container trio open/deposit/withdraw), each timeout-wrapped and AbortController-cancellable, all using `resolveBlock`/`resolveEntity` from Plan 01, all returning deterministic *what*-only result strings, and registered them in `createDefaultRegistry()` behind closed Zod schemas.

## What Was Built

### Single-target handlers (Task 1)

- **`dig.js`** — single-flight (refuses if `bot.targetDigBlock != null`), `bot.canDigBlock` pre-check, calls `bot.stopDigging()` on timeout/abort branches. Default 8 s. Returns `dug ${name}` / `cannot dig ${name}` / `busy digging` / `timeout` / `aborted` / `no target` / `stale target`.
- **`place.js`** — resolves `args.against` via `resolveBlock`, equips the named item to hand, defaults faceVector to `Vec3(0,1,0)`, races `bot.placeBlock`. Default 4 s.
- **`equip.js`** — finds inventory item by name, races `bot.equip(item, destination)`. Default 2 s.
- **`attack.js`** — resolves entity, **refuses Player targets** (`type === 'player'` or `username` truthy), enforces 3.5-block reach, single `bot.attack(entity)` swing. Default 6 s.
- **`consume.js`** — optional `args.item` equips first, **calls `bot.clearControlStates()` before `bot.consume()`** (Pitfall 3). Default 4 s.
- **`lookAt.js`** — coord (`y+1.6` eye offset) OR entity handle (`+entity.height`); empty args returns `'looked'` no-op. Default 2 s.

### Drop / activate / sleep / container (Task 2)

- **`drop.js`** — `bot.toss(itemType, null, count)`. Default 2 s.
- **`activate.js`** — `bot.activateItem()` fire-and-return. Default 2 s.
- **`sleep.js`** — **pre-checks `bot.time.isDay` and returns `'cannot sleep during day'` BEFORE calling `bot.sleep`** (Pitfall 4). Validates the resolved block is a bed (`bot.isBlockBed` if available, else `name.endsWith('_bed')`). Default 12 s.
- **`container.js`** — module-scoped `SESSION = { container, blockPos }` plus 4 exports:
  - `closeContainerSession()` — idempotent close; called by Plan 03 on chain end / abort.
  - `openContainerAction` — closes any prior session first (Pitfall 6), 4-block reach check, on success stores `SESSION.container`. Default 6 s. Cleanup on timeout/abort calls `closeContainerSession()`.
  - `depositItemAction` — requires open session, finds inventory item, races `SESSION.container.deposit`. Per-plan: does **not** close session on transfer-timeout (server-side may still settle). Default 4 s.
  - `withdrawItemAction` — requires open session, returns `'chest empty'` when item absent in chest, races `SESSION.container.withdraw`. Maps prismarine `"no room"` errors to `'inventory full'`. Default 4 s.

### Registry (Task 3)

`src/registry.js` now imports all 12 handlers and registers them in `createDefaultRegistry()` with closed Zod schemas. A reusable `TargetShape` (`block? | target? | x/y/z?` + `maxDistance` 1–64 default 32 + `.refine(...)`) is factored at module scope and reused for `dig`/`sleep`/`openContainer` and as the `against` sub-shape of `placeBlock`. `attackEntity` accepts `entity | target | entity_id` (refined: at least one). `lookAt` accepts all-optional coord-or-entity. Item ops (`dropItem`/`depositItem`/`withdrawItem`) accept `{ item, count: int 1..64 default 1 }`.

`registry.list()` now returns:
```
goTo, setGoals, dig, placeBlock, equip, attackEntity, consumeItem,
lookAt, dropItem, activateItem, sleep, openContainer, depositItem, withdrawItem
```
(14 entries — `look()` is intentionally a personality-tier tool per 2.1-RESEARCH and will be wired by Plan 03.)

## Key Decisions Made

- **`look()` deliberately not registered** in the movement-tier registry — it's a personality-tier tool (per 2.1-RESEARCH) and Plan 03's orchestrator will surface it. Keeps the movement LLM's surface to mineflayer-binding actions only.
- **`TargetShape` factored at module scope** in `registry.js` and reused by `dig`/`sleep`/`openContainer` (and as the `against` of `placeBlock`) — single source of truth for the shape `resolveBlock` consumes.
- **Container item-id source** uses `invItem.type` (mineflayer `Item.type` is the numeric id) with `mcDataLib(bot.version).itemsByName[name]?.id` as a fallback. No bespoke id table.
- **Withdraw "no room" mapping** lives in the catch branch of `withdrawItemAction` — only this op needs the inventory-full distinction; deposit/drop do not.
- **`lookAt` empty-args returns `'looked'`** rather than failing, so the personality LLM can call `lookAt({})` purely for an orchestrator-side snapshot refresh in Plan 03 without a spurious error.

## Deviations from Plan

**None substantive.** Two micro-adjustments:

1. **`lookAt`** — when an entity-shaped target (`#N` handle or `entity:name`) resolves to null, the handler returns `'stale target'` / `'no target'` instead of `'cannot look'`. Plan text specified `'cannot look'` as the catch branch only; matching dig/attack semantics for null targets is more consistent and the plan's `<done>` criteria don't constrain this case.
2. **`sleep`** — when the resolved block is not a bed, returns `'no target'` (a deterministic D-35 token already in the lexicon) rather than coining a new `'not a bed'` token. Plan didn't specify this case explicitly.

Neither changes any tested invariant.

- **No orchestrator-side snapshot injection / `last_action_result` plumbing yet** — handler return strings are correct but nothing consumes them per turn. Plan 03 wires composeSnapshot + last_action_result.
- **`buildCachedSystem` callsite mismatch from Plan 01** remains (Plan 01 already noted) — Plan 03 fixes orchestrator integration.
