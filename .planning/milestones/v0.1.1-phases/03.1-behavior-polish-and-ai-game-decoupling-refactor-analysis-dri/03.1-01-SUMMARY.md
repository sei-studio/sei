## Decisions Made

- **Comment-level cleanup of `mineflayer` mentions in brain files:** Necessary to satisfy must-have truth #3 ("No file under src/brain/ imports the string 'mineflayer'") AND the strict acceptance criterion (`grep -rl 'mineflayer' src/brain/` returns exit=1). The plan as written allowed only "import" rewrites, but the literal grep check forced touching three comments (orchestrator.js × 2, sessionState.js × 1, errStrings.js × 2). Replacements use neutral terminology (`adapter-supplied`, `adapter-side`, `adapter/runtime`) without changing semantics. Tracked as a Rule 1 fix (literal acceptance criterion would otherwise fail).
- **Comment-level cleanup of `adapter/minecraft` literal in `types.js`:** The initial draft referenced `src/adapter/minecraft/index.js` in a docstring; the plan-defined acceptance check `grep -rln 'adapter/minecraft' src/brain/` would have returned exit=0. Rewrote the comment to "the minecraft adapter under src/adapter/" preserving meaning. Tracked under deviations.
- **Task 1 split into two commits:** The first `git commit` captured only the moves (Edit tool changes hadn't been re-staged after `git add`). Rather than `git commit --amend` (which the per-CLAUDE.md guidance prefers to avoid), a follow-up commit `7df54b6` was created with a clear message linking it to Task 1. This keeps history forward-only.
- **No backwards-compat shims:** Per CLAUDE.md "Don't add backwards-compatibility hacks", no stub re-export files were created at the old paths. Build breakage is the explicit signal Plan 02 needs to find consumer-layer call sites.
