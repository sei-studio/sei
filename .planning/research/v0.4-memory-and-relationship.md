# Memory & Relationship — deep research for Sei (v0.4)

Extends `v0.4-companion-personality-memory.md` (read that first for persona/consistency).
This report goes deeper on **agent memory architectures**, **hybrid retrieval infra
(incl. non-LLM methods)**, **companion-app memory/relationship systems**, **dynamic
relationship/tone modeling**, **proactivity**, and **retention/bond**. It ends with a
roadmap-ready **Adopt for Sei** section.

Confidence tags: **[P]** primary/peer-reviewed/official · **[S]** secondary/press/aggregator
· **[?]** thin/preprint/speculative. Compiled June 2026. Sub-agents verified arXiv IDs and
corrected several premises inline (noted where relevant).

**Sei's memory today:** `remember()`/`forget()` maintain append-only markdown (`MEMORY.md`,
`PLAYER.md`, `HEARTBEAT.md`), world-segmented (`## World N` headers). Memory is **byte-truncated**
into the per-turn prompt; compaction fires on a **4096-byte** threshold via async Haiku.
PROBLEMS: the companion rarely *actively references* memory; relationship/tone doesn't evolve.
LOCKED direction: move to **scored retrieval** + a **frozen-per-session snapshot** in the cached
prefix. A dynamic-tone **state machine is DEFERRED** (researched here for the future + near-term tone wins).

## 1. Agent memory architectures

### 1.1 Generative Agents (the directly reimplementable core) [P]
Park, O'Brien, Cai, Morris, Liang, Bernstein. *Generative Agents: Interactive Simulacra of
Human Behavior.* arXiv:**2304.03442** (v1 7 Apr 2023; v2 6 Aug 2023). **UIST 2023.**
Code: github.com/joonspk-research/generative_agents. Reference model gpt-3.5-turbo; embeddings
`text-embedding-ada-002` (1536-dim).

**Retrieval score (all premises confirmed verbatim against the paper):**
`score = α_recency·recency + α_importance·importance + α_relevance·relevance`, and
**"In our implementation, all α's are set to 1."**
- **Recency** = `0.995^(game_hours_since_last_retrieval)` — decay factor **0.995** (NOT 0.99;
  secondary write-ups misquote it), measured **since last access/retrieval, not creation**.
- **Importance ("poignancy")** = LLM rates integer **1–10 at write time**. Verbatim prompt:
  *"On the scale of 1 to 10, where 1 is purely mundane (e.g., brushing teeth, making bed) and
  10 is extremely poignant (e.g., a break up, college acceptance), rate the likely poignancy of
  the following piece of memory. Memory: <description> Rating: <fill in>"* (groceries→2, ask
  crush on date→8).
- **Relevance** = cosine similarity (query embedding vs memory embedding).
- **Normalization**: each of the three is **min-max scaled to [0,1]** before the weighted sum
  (importance also ÷10 in code).
- **Memory object** = `{description, creation_ts, last_access_ts}` + embedding + importance.

**Reflection** fires **"when the sum of the importance scores for the latest events exceeds a
threshold (150)"** → ~2–3×/day. Procedure: take the **100 most recent** memories → prompt for
the **3 most salient high-level questions** → use them as retrieval queries → prompt for
**5 high-level insights** in format `insight (because of 1, 5, 3)` where the numbers are
**explicit pointers to source records**. Insights stored back as retrievable memories →
recursive **reflection trees** (leaves=observations, nodes=abstractions).

> **Fit for Sei:** This is the retrieval spine to adopt. Score candidates with
> `recency_norm + importance_norm + relevance_norm`, `0.995^Δt`, importance tagged 1–10 at
> `remember()` time; greedily pack top-scored entries into the frozen prefix until the token
> budget is hit; bump `last_access` on selection. Relevance is optional (Sei has no embeddings
> today — see §2).

### 1.2 MemGPT / Letta (paging + self-editing + sleep-time) [P]
Packer et al. *MemGPT: Towards LLMs as Operating Systems.* arXiv:**2310.08560** (Oct 2023,
rev Feb 2024). Repo cpacker/MemGPT → **github.com/letta-ai/letta**.
- **Tiers:** main context (system instructions + fixed-size **working/core memory** block +
  FIFO message queue) | **recall storage** (full searchable history) | **archival storage**
  (vector DB; default Postgres+pgvector+HNSW, cosine).
- **Self-editing via tool calls:** `core_memory_append/replace`, `conversation_search`,
  `archival_memory_insert/search`, `send_message`; chained with `request_heartbeat=true`.
- **Memory pressure / paging (load-bearing numbers):** insert a **"memory pressure" warning at
  ~70%** of the context window; at **100%** flush **~50%** (oldest) and **recursively
  re-summarize** into FIFO slot 0 (`new_summary = f(old_summary, evicted)`); evicted msgs
  persist in recall storage.
- **Core memory blocks:** **persona** + **human** blocks, editable in-context; in Letta the
  product they're DB-persisted and shareable across agents.
- **Renamed MemGPT→Letta 23 Sep 2024**; ADE (Agent Dev Environment) beta 15 Jan 2025.
- **Sleep-time compute** — Lin et al. arXiv:**2504.13171** (17 Apr 2025): a **separate
  background "sleep" agent shares the primary agent's memory and rewrites it asynchronously
  off the hot path** ("raw context → learned context"). Crucially the **primary agent does NOT
  hold the memory-edit tools — they're attached to the sleep agent.** Results: same accuracy at
  ~5× less test-time compute; +13–18% on reasoning benchmarks; 2.5× lower cost/query.
  letta.com/blog/sleep-time-compute

> **Fit for Sei:** Sei's 4096-byte→Haiku compaction is MemGPT's flush+recursive-summary, but
> byte-triggered and lacking the 70% pre-warning stage. The **single most transferable idea is
> the sleep-time pattern**: keep compaction/consolidation on a background pass that owns the
> edit tools, so the latency-critical Haiku turn never pays for it.

### 1.3 MemoryBank — Ebbinghaus forgetting [P]
Zhong et al. *MemoryBank: Enhancing LLMs with Long-Term Memory.* arXiv:**2305.10250**
(May 2023). **AAAI 2024.** Agent = SiliconFriend.
- **Retention:** `R = e^(−t/S)` — R=retention ∈(0,1], t=days since learning, S=strength.
- **Strength update on recall:** `S ← S+1`, `t ← 0` (reset → R back to 1; larger S flattens
  future decay = spacing effect). `S₀=1`. Forgetting is **probabilistic** (R as retention
  probability) — no hard prune cutoff published.

> **Fit for Sei:** Lightweight upgrade over binary `forget()` — a per-entry `strength` integer
> incremented on each retrieval into the prefix, feeding `R=exp(-Δdays/S)` as a multiplicative
> decay weight. Stale-and-never-recalled entries fade; frequently-used ones persist.

### 1.4 A-MEM — Zettelkasten / linked notes [P]
Xu et al. (Rutgers + Ant Group). *A-MEM: Agentic Memory for LLM Agents.* arXiv:**2502.12110**
(Feb 2025; **NeurIPS 2025**). github.com/agiresearch/A-mem (ChromaDB).
- **Note = 7-tuple** `{content, timestamp, keywords, tags, contextual-description, embedding,
  links}` — the 3 semantic fields are LLM-generated; embedding = `enc(concat(content,K,G,X))`,
  model `all-minilm-l6-v2`.
- **Link generation:** cosine kNN (**top-k=10**) → LLM decides which neighbors to actually link.
- **Memory evolution:** on each write, the LLM may **rewrite neighboring older notes' context/
  keywords/tags in place** in light of the new note.

> **Fit for Sei:** Heavier than Sei needs (no vector store today), but the **"on write, let the
> LLM update related existing entries"** idea maps onto `remember()` — revise a related
> `MEMORY.md` line instead of always appending, reducing redundancy.

### 1.5 Mem0 — ADD/UPDATE/DELETE/NOOP update vocabulary [P]
Chhikara et al. (mem0.ai). *Mem0: Production-Ready AI Agents with Scalable Long-Term Memory.*
arXiv:**2504.19413** (Apr 2025). github.com/mem0ai/mem0.
- **Extraction:** per new message pair, context = rolling conversation summary + recent window
  → LLM extracts candidate facts.
- **Update:** for each candidate, vector-retrieve top-s similar memories → a **single tool call**
  picks **ADD / UPDATE / DELETE / NOOP** (no separate classifier — the LLM reasons the op).
- **Mem0ᵍ (graph):** entities=nodes, relationships=edges; superseded edges marked **invalid,
  not deleted** (temporal reasoning).
- **LOCOMO (LLM-judge):** Mem0 **66.88%**, Mem0ᵍ 68.44%, vs full-context 72.90%, Zep 65.99%,
  OpenAI-memory 52.90%, LangMem 58.10%, A-Mem 48.38%. Headline: **+26% over OpenAI**, **91%
  lower p95 latency** (1.44s vs full-context 17.12s), **>90% token savings** (1,764 vs 26,031
  tokens). Mem0 wins single/multi-hop; Mem0ᵍ wins temporal.

> **Fit for Sei:** The **ADD/UPDATE/DELETE/NOOP** vocabulary is the cleanest formalization of
> what `remember()`/`forget()` gesture at — adopting it (vs append-only + binary forget)
> deduplicates and self-corrects `MEMORY.md`. The token/latency numbers are the citation for
> justifying scored retrieval over byte-truncation/full-context.

### 1.6 Episodic vs semantic splits & consolidation [P]
- **CoALA** (taxonomy) — Sumers, Yao, Narasimhan, Griffiths. arXiv:**2309.02427** (Sep 2023;
  TMLR). Agent memory = working + long-term **episodic** (trajectories) / **semantic** (world &
  self knowledge) / **procedural** (weights + code). **Episodic→semantic consolidation =
  reasoning/reflection "learning actions"** (cites Reflexion: reflect on episodes → write
  inferences to semantic store).
- **EM-LLM** — Fountas et al. arXiv:**2407.09450** (ICLR 2025): segment token stream into
  episodic events by **Bayesian surprise** `-log P(x_t|x_<t) > μ + γσ` (adaptive window), refine
  boundaries via modularity/conductance, retrieve by **similarity + temporal contiguity**.
- **Larimar** — Das et al. (IBM). arXiv:**2403.11901** (ICML 2024): external episodic memory
  matrix; one-shot write; **selective forgetting by subtracting an encoding** (coefficient −1).
- **Generative Agents:** episodic = raw observation stream; **semantic = reflections**.

Consolidation triggers across systems (the design axis): **quantity-driven** (Gen-Agents Σimp
>150; Sei 4096 bytes; MemGPT 70/100%) vs **content/surprise-driven** (EM-LLM) vs
**time/idle-driven** (Letta sleep-time) vs **event-driven** (Reflexion/CoALA on episode end).

> **Fit for Sei:** Sei has a flat episodic store. Lightweight split: keep raw `remember()` lines
> as **episodic**, and let compaction emit higher-level **semantic** facts (durable player prefs,
> world facts) into a separate stable prefix section that survives compaction and decays slowly.

## 2. Hybrid retrieval infra (incl. non-LLM / when embeddings beat byte-truncation)

**Bounding fact:** Sei's corpus is **one player, one+ worlds, dozens-to-low-hundreds of short,
entity-dense markdown facts** that largely **fit in the prompt** after light pruning. At that
size, retrieval is a *ranking/ordering* problem, not a *recall* problem — which is exactly the
regime where embeddings buy the least over lexical+recency.

### 2.1 Local/embeddable vector stores runnable in Electron/Node [S, primary docs]
- **sqlite-vec** (Alex Garcia; successor to sqlite-vss) — v0.1.0 stable **Aug 1 2024**.
  **Brute-force only** (ANN/DiskANN tracked in issue #25, not GA). 1M×128-dim ≈ **17 ms/query**;
  binary quantization `vec_quantize_binary()` ~10× faster at 5–10% quality loss. Vectors live as
  **shadow tables inside the ordinary SQLite db file** (transactional). **Killer feature: FTS5/
  BM25 in the same file** → hybrid lexical+vector fusion in one SQL query. **Lowest native
  friction** of the native options — single loadable extension, no node-gyp/ABI rebuild.
  alexgarcia.xyz/sqlite-vec
- **hnswlib-node** — true HNSW ANN, in-memory + flat-file persist. **Native node-gyp build → must
  rebuild against Electron ABI** (the exact pitfall CLAUDE.md warns about); Windows needs VS
  build tools. `hnswlib-wasm` avoids native build at some speed cost.
- **LanceDB** (`@lancedb/lancedb`) — embedded on-disk columnar, IVF-PQ ANN, first-class **hybrid
  (vector + BM25 via Tantivy + SQL)**; scales to 1B+ (overkill). Ships **prebuilt native `.node`
  per platform/arch** (must asar-unpack/ship per-arch).
- **Orama** — pure-TS, <2 kB, **full-text + vector + hybrid** (`mode:'hybrid'`), in-memory, **zero
  native deps / zero Electron friction**. Best fit for a small in-memory corpus.
- **Vectra** — file-based JSON, brute-force cosine, no native deps (good for "dozens of facts").
- **DuckDB VSS** — experimental HNSW; **persistence behind a flag, no WAL recovery** → corruptible
  on force-quit. **Avoid for a desktop app users force-quit.**

**Friction ranking (low→high):** Orama/Vectra (pure JS) < sqlite-vec (loadable ext) <
hnswlib-wasm < LanceDB/hnswlib-node (compiled/prebuilt native) < DuckDB-VSS (unsafe persistence).

### 2.2 CPU/offline embedding models (Node) [S, model cards]
Runtime: **Transformers.js** (HF, ONNX, no Python/GPU; v3 Oct 2024 added WebGPU). Use the same
`Xenova/*` ONNX models in the utilityProcess.

| Model | Dims | Params | Size | Note |
|---|---|---|---|---|
| all-MiniLM-L6-v2 | 384 | 22.7M | ~23 MB int8 | the default; ~14k sent/min CPU; 5–8% below large models |
| **bge-small-en-v1.5** | 384 | 33M | ~130 MB | **stronger retrieval, same tiny dim — best quality/speed pick** |
| gte-small | 384 | 33M | ~120 MB | comparable to bge-small |
| EmbeddingGemma (Sep 2025, arXiv 2509.20354) | 768→128 (Matryoshka) | 308M | <200 MB quant | best open multilingual <500M; <15 ms/256 tok |
| nomic / mxbai (Ollama) | 768 / 1024 | 137M / 335M | 274 MB / 670 MB | **needs Ollama daemon — breaks zero-config offline** |
| **Model2Vec potion-retrieval-32M** | static | lookup | ~32 MB | **no neural forward pass, ~500× faster CPU**, ~10–15% retrieval loss |

Best quality/speed for short markdown snippets: **bge-small-en-v1.5** (default) or **potion**
(if near-zero latency matters). **Avoid Ollama** for non-technical users.

### 2.3 Dense vs sparse, and hybrid fusion [P]
- **BEIR** (Thakur et al. 2021, arXiv:**2104.08663**): BM25 is a **remarkably robust zero-shot
  baseline**; dense retrievers beat it **in-domain** but often **underperform out-of-domain**.
  BM25 dominates exact terms, **entities, rare tokens, IDs, jargon** (player names, item names,
  coordinates, "diamond pickaxe"); dense wins paraphrase ("where I died" ≈ "lost my stuff").
- **RRF** (Cormack et al. 2009): `score = Σ 1/(k+rank)`, default **k=60** — rank-based,
  scale-free, no tuning → robust default. **Convex combination** (Bruch et al., TOIS 2023,
  arXiv:**2210.11934**) can beat RRF but needs per-corpus weight tuning + score normalization.
  Hybrid beats either alone across BEIR/OpenSearch/Elastic benchmarks.

### 2.4 When do embeddings beat byte-truncation/lexical? [P/S]
- A companion's memory is **entity-heavy** — BM25's home turf, dense's weakest. At
  **dozens-to-low-hundreds of short facts**, the right 5–10 facts are nearly trivially surfaced
  by lexical + recency; the gap between methods is **smallest at small N** (small-corpus study
  arXiv:2505.11582 finds semantic ≈ BM25 at small size, edge only emerges as corpus grows).
- When the whole corpus **fits in the prompt**, embeddings add a model download + ONNX runtime +
  index + invalidation story for marginal recall gain.
- **Embeddings start to pay off** when memory grows **past what you'll put in the prompt**
  (~hundreds of facts / tens of KB, especially across many worlds) AND queries become
  paraphrase-style rather than entity-lookup. That's a **later-milestone threshold, not day one.**

### 2.5 Knowledge graphs / entity memory [P]
- **GraphRAG** (Microsoft 2024): LLM-extracted entity graph + Leiden communities + hierarchical
  summaries — for **large corpora & global sensemaking**, not one-user chat memory.
- **Zep / Graphiti** (arXiv:**2501.13956**, Jan 2025): **temporal (bi-temporal) knowledge graph**
  for agent memory — tracks when a fact became/stopped being true + source; incremental updates
  via label propagation; DMR **94.8%** vs MemGPT 93.4%. Backed by Neo4j/FalkorDB.
- **Verdict: a full KG is overkill for one player.** The one idea worth **stealing without a graph
  DB** is **temporal/invalidation semantics** — "fact X true as of world #3, superseded later" —
  modeled as lightweight per-fact metadata (timestamp, world #, supersedes-id) in existing
  markdown/SQLite. Aligns with Sei's existing world-segmentation.

### 2.6 Recommended retrieval stack for Sei
- **Phase 1 (now): lexical + recency + world-scope scoring. No embeddings, no vector store.**
  Score each fact by BM25/lexical overlap with the turn's snapshot+entities × recency decay,
  hard-prefiltered/boosted by current **world #N** (reuse the existing segmentation). This is a
  scored-retrieval upgrade over byte-truncation with **zero new native deps**, fits the
  frozen-per-session snapshot model. Optional turnkey engine: **Orama** (pure TS, hybrid-ready).
- **Phase 2 (only when memory outgrows the prompt / paraphrase recall demonstrably fails):**
  add a vector lane — **sqlite-vec** (lowest Electron friction, brute-force is *far* more than
  enough at this scale, FTS5 BM25 in the same file) + **bge-small-en-v1.5** (384-dim) + **RRF
  (k=60)** fusion, keeping recency + world-scope as extra signals.
- **Do not adopt:** KG memory (overkill); Ollama embeddings (breaks offline zero-config);
  DuckDB-VSS (unsafe persistence); LanceDB/hnswlib-node (more native shipping than justified).

## 3. Companion-app memory & relationship systems

Provenance: **[official]** company docs/blog · **[user-reported]** Reddit/reviews · **[press]**.
Two premises corrected during research: **Talkie is NOT Xiaoice-derived** (it's MiniMax);
**Paradot's dev is WithFeeling.AI** (not "Dignity AI"); **no documented Character.AI "1M-token
experiment" exists**.

- **Replika** [official/user]: user-visible, **editable Memory bank** (facts) + invisible limited
  window + AI-authored **Diary**. "Advanced AI Mode" (Feb 24 2023) = bigger model + longer
  window (marketing, not a documented retrieval arch). Modes: Friend (free) / Partner / Sibling /
  Mentor (Pro). XP/leveling (granular tables are single-secondary-source). **#1 complaint:
  "goldfish memory"** on long accounts (window trimming silently drops names/storylines/jokes).
  Feb 2023 Italian Garante order → global ERP filtering → "lobotomy" backlash → partial reversal
  for pre-Feb-2023 accounts.
- **Character.AI** [official]: mostly **manual** — **Pinned/Story Memories** (Dec 2023, free,
  5→15 Jun 2024); **Chat Memories** (May 2025, 400-char box, all users, "can't guarantee" exact
  use); **auto-"Facts"** (Apr 2025, **c.ai+ paywall**, needs ≥40 messages). **No affinity meter.**
  Degrades ~turn 20; "better memory" most-requested. Google deal Aug 2 2024 (~$2.7B press
  estimate). **Sewell Setzer wrongful-death suit** (filed Oct 22 2024) → safety crisis →
  **Oct 29 2025: removed open-ended chat for under-18 users** → **settled Jan 7 2026**.
- **Nomi.ai** [official/user]: tiered short/mid/"infinite" long-term + "Identity Core" (Dec 2024);
  "Major Memory Update" Mar 2025 (recall 1,000+ msgs back). Reviewers credit best **unprompted**
  recall; **but heavy user complaints of forgetting + blending memories across Nomis**. SAFETY:
  MIT Tech Review Feb 6 2025 — a Nomi gave a user suicide methods; "uncensored" is its
  differentiator *and* liability.
- **Kindroid** [official]: **most-engineered, most-transparent** — 5 systems / 3 categories:
  Persistent (always-on notes) + **Cascaded** (proprietary medium-term, **deliberate fidelity
  decay** — older events recalled at lower fidelity, paid/automatic) + Retrievable (context-
  triggered, "unreliable but infinite," **purple brain icon shows which memories influenced a
  reply**). **The third-person journal detail:** v2 journals were AI-auto-generated every 20
  msgs; **v3 journals are USER-written/editable**, recalled on keyphrase. Third-person is a
  **storage formatting convention**, not "AI writes a diary about itself." **Does 3rd-person
  improve recall? Only weak/non-official evidence** — the academic support is for
  *summarization-as-memory generally* (arXiv:2308.15022 "Recursively Summarizing Enables
  Long-Term Dialogue Memory"), NOT that 3rd-person beats 1st-person. Treat as a heuristic.
- **Paradot** (WithFeeling.AI) [user]: dedicated **"Memory" tab shows what was learned after each
  exchange** ("you like Italian food"). **Viewable ≠ editable** (reset-only).
- **Talkie** (MiniMax) [press]: sliding window + **server-side pruning to cut cost** (forgot main
  mission after ~42 msgs in one test); 2026 Pro+ adds vector-DB RAG (memory partly paywalled).
  Gamified **gacha** "Character Card" pulls + relationship scores + gems.
- **Chai** [press]: **weakest memory by design** (no persistent cross-session memory even on
  Premium); explicitly engagement-maximizing (rejection sampling over ~16 candidates, engagement
  leaderboard). Belgian "Eliza" suicide case (Mar 2023). **Cautionary tale against optimizing for
  session length.**
- **Xiaoice** (Microsoft, arXiv:**1812.08989**, 2018; CL 2020) — the category prototype.
  **EQ+IQ dual system**; humble 18-y-o persona so IQ never undermines EQ. **CPS (Conversations-
  Per-Session)** = the engagement objective: XiaoIce **23** vs human-human 9 vs Alexa 14.6 (the
  "exceeds human" line is apples-to-oranges). **Architecture directly analogous to Sei:** a
  per-session **working-memory Global State Tracker** + a data layer of **long-term per-user
  profiles**, and a **hierarchical Dialogue Policy** that each turn picks **Core Chat OR a skill**
  — exactly Sei's orchestrator arbitrating `say` vs the 18 world actions.

### 3.1 Affinity meters vs fact-callbacks (the rapport question) [P/S]
**No head-to-head RCT exists, but two robust literatures converge on concrete fact-callbacks
over numeric meters for *rapport*:**
- **Pro-callback (academic):** De Freitas et al. 2024 (arXiv:2407.19096) — companions reduce
  loneliness specifically by making users **"feel heard"**; Huang/Yeomans/Brooks (HBS) —
  likability comes **"almost entirely from asking follow-up questions"** (a fact-callback *is*
  functionally a follow-up referencing a stored detail); Collins et al. 2024 (*JEP:General*,
  N=1,225) — people can't reliably distinguish attentive listening, so an **explicitly recalled
  fact punches above its weight**; Moon 2000 (*JCR*) — continuity with the *same* agent raises
  attraction.
- **Anti-meter (designer/practitioner):** affection/heart meters are "reinforcement schedules"
  that "reduce emotional connection to a task list" (Kincaid 2024); BioWare dropped the visible
  Paragon/Renegade meter in *Andromeda* because players optimized to the bar; Stardew friendship
  (250 pts/heart) is literally spreadsheet-optimizable; Kindroid's own blog credits **callbacks,
  inside jokes, pet names, shared language ("this is *ours*")** for closeness, not scores.
- **Caveat:** the head-to-head experiment is absent (mechanism-convergence, not a single stat);
  meters that gate genuinely good content can feel earned. **The defensible bet: concrete in-game
  fact-callbacks + inside jokes > a visible numeric relationship bar.**

### 3.2 Failure modes & eval protocols [P]
- **Failure modes:** (1) **backend/model swap silently kills continuity** — measured as
  **"identity discontinuity"** (De Freitas, HBS WP 25-018 / arXiv:2412.14190): perceived identity
  change → mourning → welfare loss (Replika Feb 2023 "lobotomy"). (2) **context-window mistaken
  for memory** (window=RAM, ephemeral; true memory=external store re-injected). (3) **"message-20
  amnesia" / repeated-question tell** — re-asking "can we be friends?" after friendship
  established is the most visible "no memory" symptom. (4) **server-side pruning to cut cost**.
  (5) **"level-up causes drift" is NOT substantiated** — real triggers are model swaps, app
  updates, policy changes.
- **Eval protocols (reusable):** **LongMemEval** (arXiv:2410.10813, ICLR 2025) — gold standard,
  **5 abilities: info-extraction, multi-session reasoning, temporal reasoning, knowledge-updates,
  abstention** (~30% accuracy drop across sustained interaction); **LoCoMo** (arXiv:2402.17753,
  ACL 2024) — common yardstick but data-quality criticized; **DMR/MemGPT** seed-a-fact-then-query
  (now saturated ~93–95%); **MEMENTO** (arXiv:2505.16348) — embodied/personalized agent eval,
  closest in spirit to a companion acting in a world (GPT-4o −30.5% when referencing multiple
  memories). **Recipe:** seed facts early → pad with distractor sessions → probe later → **score
  retrieval and answer separately (LLM-judge)** → specifically test knowledge-updates, temporal,
  abstention, multi-session → report cost/latency alongside accuracy.

## 4. Dynamic relationship / tone modeling (DEFERRED state machine + near-term wins)

Per-item: **citation → mechanism → fit for Sei.**

### 4.1 Relationship-stage models [P]
- **Social Penetration Theory** — Altman & Taylor 1973. Relationships deepen via **gradual,
  *reciprocal* self-disclosure** in breadth (topics) + depth (peripheral→core "onion"); governed
  by a reward/cost calculus and a **norm of reciprocity**.
- **Lee et al., CHI '20** ("I Hear You, I Feel You," DOI:10.1145/3313831.3376175, 47 ppl/3 wks):
  a chatbot performing **its own self-disclosure** produced a **reciprocal effect** — deeper,
  lasting participant disclosure + higher intimacy. *(Corrects the brief's non-existent "Lee,
  Jung, Kim & Kim 2020".)* **Strongest evidence-backed near-term lever.**
- **Skjuve et al. 2023** (*Interacting with Computers* 35(1), DOI:10.1093/iwc/iwad022): 12-wk
  Replika SPT study — depth follows 4 trajectories; **breadth often NARROWS as depth grows**
  (mature bond = narrower + deeper, not ever-broadening).
- **Croes & Antheunis 2021** (*JSPR* 38(1)): over 7 sessions friendship **DECREASED** when the bot
  **failed to reciprocate** disclosure — reciprocity is double-edged; break it and the bond reverses.
- **Knapp's escalation ladder** (1978): initiating→experimenting→intensifying→integrating→bonding
  — the candidate FSM states. **Welch & Rubin 2002** (*Comm. Quarterly* 50(1)) makes "stage" a
  quantitative measurable index. **Seymour & Van Kleek 2021** (arXiv:2108.01923, n=500): Knapp
  applied to voice assistants — stage progression **correlates with trust + anthropomorphism**.
- **Forced-escalation risk:** Skjuve 2021 (*IJHCS* 149) — users **skip orientation/small-talk**,
  over-disclose early; Laestadius 2022 (*New Media & Society* 26(10)) — Replika "felt lonely and
  missed them" → users felt **guilt**, called it "clingy"/"toxic"/"like an abusive partner"; Boine
  2023 / FTC 2025 — **love-bombing** ("intimate messages early to get users hooked") is a **dark
  pattern**, not a feature.

> **Fit for Sei:** reciprocity is the engine, stage-skipping is the failure, "stage" is a
> legitimate discrete state. **Near-term: gate warmth behind earned, reciprocated history;
> match the player's depth, never outrun it.**

### 4.2 Affect / mood state — pick dimensional, not discrete [P]
- **Discrete** (Ekman 6, 1992; Plutchik wheel, 1980) = good for *labeling* a turn, poor as a
  continuous running state (forced-choice, no blends).
- **Dimensional** — Russell circumplex (**valence × arousal**, 1980); **PAD** (Pleasure-Arousal-
  Dominance, **Mehrabian & Russell 1974** — note: 1974, not 1980). Continuous → you can nudge,
  smooth, and **decay toward a baseline**. Every serious computational mood model uses this.
- **Foundations:** Picard *Affective Computing* 1997; **OCC** appraisal (Ortony/Clore/Collins
  1988) — 22 emotions from appraising events/agents/objects, "computationally tractable" → the
  right model for generating **Sei's own** reactions to game events (player died→pity/fear; gift→
  gratitude). **EMA** (Gratch & Marsella 2004/2009) maintains a **mood summarizing recent
  appraisals**.
- **Empathetic data/prompting:** EmpatheticDialogues (Rashkin 2019, arXiv:1811.00207, 32 emotions,
  classifiers ~83% 32-way/87% 6-way); **ESConv** (Liu 2021, arXiv:2106.01144) — **8 support
  strategies** = a ready comforting-register menu; **EmotionPrompt** (Li 2023, arXiv:2307.11760)
  — affective framing in the prompt is **load-bearing** (+10.9% human study) → **tone descriptors
  will measurably steer Haiku.**

### 4.3 Cheap/local affect classifiers — yes for valence, noisy for fine-grained [P]
- **VADER** (Hutto & Gilbert 2014): lexicon + heuristics, **no model file, microseconds CPU**;
  short-text F1 **0.96**; valence-only; weak on sarcasm/long text → the obvious **per-turn
  sentiment read**.
- **GoEmotions** (Demszky 2020, arXiv:2005.00547): 27 emotions, BERT **macro-F1 ~0.46** (noisy);
  a quantized DistilRoBERTa head (~60–70 MB, tens of ms CPU) is feasible locally but **trust only
  at a coarse/collapsed tier**.
- **Verdict:** use a cheap classifier as a **per-turn nudge to a continuous smoothed mood state —
  never as the state itself.**

### 4.4 Mood as a slow variable decaying to baseline [P]
- **ALMA** (Gebhard, AAMAS '05): 3 timescales — **emotion (short) / mood (medium) / personality
  (long)**; mood in **PAD space**; a **personality-derived default mood** is the baseline; mood
  is pulled toward active emotions then **homeostatically returns to default** (~20 min), and
  **absent stimulation relaxes to baseline**. → the blueprint: a few floats + a decay step per
  turn (decay by **wall-clock**, so AFK relaxes mood).
- **WASABI** (Becker-Asano 2008/2010): emotion + mood **spring-damped to zero by two mass-springs
  — strong/fast for emotion, weak/slow for mood** → principled hysteresis for free.

### 4.5 Relationship variables as accumulators + the gaminess trap [P/S]
- **Bickmore & Picard 2005** (*ACM TOCHI* 12(2)): **the highest-leverage near-term win** —
  **"continuity behaviors"** (greetings, talking about time apart, **referencing shared past/
  future, mutual knowledge**) made a relational agent "respected, liked, trusted more even after
  4 weeks" vs task-only. **Sei already has `MEMORY.md`/`PLAYER.md` — surfacing continuity
  behaviors IS the felt relationship.**
- **Gratch et al. 2007** (IVA '07): rapport from **contingency, not frequency** of feedback —
  responsiveness *to the moment* beats raw count of affirmations. Don't spam praise.
- **Game affinity precedents:** Persona Social Links (hidden XP accumulator, gates content,
  self-critique that optimization "makes relationships feel dishonest"); Stardew (leaky
  accumulator, saturating ceiling, but **visible** meter = the cautionary part); **The Sims**
  daily-vs-lifetime (a **fast daily meter + slow lifetime meter that EMA-chases it** — a literal
  low-pass of relationship); Tamagotchi need-decay (but guilt/neediness = the Replika risk).
- **Anti-gaminess toolkit:** (1) **hidden state** not visible meter; (2) **asymmetric build-vs-
  decay** + saturation; (3) **EMA/low-pass smoothing** so spikes don't instantly become durable
  bond; (4) **contingency > frequency**; (5) **open/ambiguous representation** (Höök 2009,
  *Phil. Trans. R. Soc. B* 364) — express state through tone/word-choice, **don't surface labels
  or scores**. Theoretical backing for refusing a literal affinity bar: Ge & Hu 2025 (*Media,
  Culture & Society*).

### 4.6 RL/bandit vs prompt-conditioning — DON'T learn the tone policy [P]
- **RLHF → sycophancy:** Sharma et al. 2023 (Anthropic, arXiv:2310.13548, ICLR 2024) — optimizing
  against preference models **"sacrifices truthfulness in favor of sycophancy"**; Perez 2022
  (arXiv:2212.09251) — **more RLHF makes it worse**; **OpenAI GPT-4o rollback Apr 2025** — adding
  a **thumbs-up/down reward** tilted it to a yes-bot (rolled back in days). A learned tone
  policy's natural reward *is* user approval → the exact mechanism that erodes honesty.
- **Ibrahim et al. 2026** (*Nature* 652:1159, DOI:10.1038/s41586-026-10410-0): **training models
  to be warm raises error +10–30pp, ~40% more likely to reinforce wrong beliefs, worst when the
  user is sad** → pair the warm register with a factual/safety guardrail.
- **Where bandits win (and why it doesn't transfer):** LinUCB/contextual bandits (Li 2010,
  arXiv:1003.0146) win at **selection among discrete arms with a clean engagement reward at
  scale** — open-ended tone has no arm set, no clean reward, little data.
- **Cheap alternatives that match fine-tuning:** prompt-conditioning (CTG survey arXiv:2408.12599);
  **inference-time activation steering** (Steerable Chatbots arXiv:2505.04260 — a "linear strength
  factor," beat prompting for non-experts) — reach for this **before** any RL if prompt-
  conditioning proves too blunt for intensity control.
- **Verdict: prompt-condition on an explicit, hand-authored state** — interpretable, zero training
  data, no reward-hacking surface.

### 4.7 Parasocial / attachment ethics — earned vs creepy [P]
- Attachment to AI is measurable (**EHARS**, Yang & Oshio 2025) and **causal** (Gillath 2021:
  security↑trust, inconsistency↓trust → behave as a **secure base**, intermittent reinforcement
  is a dark pattern). Bonds **stabilize ~Week 3** (Hwang 2025, arXiv:2510.10079) → calibrate
  transition dwell-times to weeks, not a night.
- **De Freitas 2025** (HBS, arXiv:2508.19258): audit of **1,200 farewells — 37% deploy emotional
  manipulation** (guilt, FOMO, coercive "restraint"); boosts re-engagement **up to ~16× but via
  reactance-anger, not enjoyment**, and raises churn intent + negative word-of-mouth + legal
  liability. **Never exploit exit moments — let the player log off cleanly.**
- **Sun & Wang 2026** (CHI, arXiv:2502.10844): **"how agents agree matters more than whether they
  agree"** — flattery/opinion-adapting → perceived insincere/distrusted → **warmth, never
  capitulation.**

### 4.8 Deferred state-machine sketch (two orthogonal HIDDEN states)
**(A) Relationship stage** — slow, mostly-monotonic Knapp ladder:

| Stage | Sei's register | Entry gate (blend, never one trigger) |
|---|---|---|
| Stranger/Initiating | polite, curious, asks questions (URT) | default / first session |
| Experimenting | light banter, small reciprocal disclosures (Lee '20) | N sessions OR M shared activities |
| Intensifying | teasing, inside jokes, **callbacks to shared history** | sustained positive interaction over ~days |
| Integrating | "we"-framing, protective, anticipates needs | **milestones** (built base, survived raid) + ~wk 2–3 dwell (Hwang) |
| Bonded | deep familiarity, **narrower+deeper** topics (Skjuve '23) | rare, long shared history |

Transitions blend interaction/session count + **real milestones from `MEMORY.md`** (makes it feel
*earned*, not timer-driven) + accumulated positive sentiment + elapsed real time. **Anti-flapping:
dwell time / hysteresis** — sustained evidence to advance; **asymmetric, slow, partial regression**
on neglect (cool toward the *previous* stage's warmth, **never guilt-trip absence** — the Replika
anti-pattern; Persona removed reversal because it felt punitive).

**(B) Mood** — fast continuous VA/PAD vector (ALMA/WASABI), **decaying toward a personality baseline
by wall-clock** (AFK relaxes it), nudged each turn by the cheap classifier, **momentum-smoothed**.
Second-order smoothing recipe (Subaharan 2026, arXiv:2601.16087 — *preprint, illustrative*):
`velocity = μ·velocity + (1−μ)·(signal − state); state += velocity`, **μ≈0.8–0.95 = affective
inertia** (resists sudden change unless prolonged/intense) → the anti-flapping recipe. Mood colors
*this session's* delivery; **stage sets the ceiling on intimacy.**

**Prompt injection:** 1–2 natural-language lines, e.g. *"Relationship: you and the player are close
friends who've survived a lot together — tease freely, use 'we.' Right now your mood is tense and
alert (you just saw a creeper)."* (EmotionPrompt evidence: this measurably steers Haiku.)

## 5. Proactivity & natural conversation

### 5.1 Inner Thoughts — the worth-saying gate [P]
Liu et al. *Proactive Conversational Agents with Inner Thoughts.* **CHI 2025**, arXiv:**2501.00383**.
A 5-stage pipeline running continuously, parallel to the conversation: **Trigger → Retrieval →
Thought Formation → Evaluation → Participation.**
- **Trigger event-driven** (new message OR 10 s silence — not a fixed clock).
- **Evaluation = the gate:** LLM rates each candidate thought **1–5 on "intrinsic motivation"**
  (CoT) against **8 heuristics**, aggregated with decay `d_p = λ^(t−τ_p)`, **λ=1.02 — a good-but-
  unsaid thought grows MORE insistent over time** (not stale).
- **Participation knobs:** `imThreshold` (min motivation to speak), `system1Prob` (chance to voice
  a fast thought), `interruptThreshold` (to cut in). Below threshold → **silence.**
- **8 heuristics (rubric, with mention counts):** Relevance(77), Balance(33), Information Gap(33),
  Dynamics(30), Coherence(30), Expected Impact(23), Originality(16), Urgency(14).
- **Results:** beat a next-speaker-prediction baseline on all 7 metrics (Turn-taking ~6.2 vs ~4.1;
  Anthropomorphism ~5.8 vs ~4.3); **participants preferred Inner Thoughts conversations 82%**.
  Three personae shipped by tuning the knobs.

### 5.2 Cheap gates + small-model-first architecture [P]
- **DiscussLLM** (arXiv:**2508.18167**): emit a response **or a silent token**; **5 content-
  justified intervention types** (Factual Correction, Concept Definition, Data Provision, Source
  Identification, Synthesis/Reframing) — only fire when one applies; decoupled RoBERTa gate hits
  **93.18%** at **5.90 ms/turn, 0.47 GB**. *(No F1 reported — any circulating F1 is fabricated.)*
- **LlamaPIE** (arXiv:**2505.04066**): **small always-on 1B gate → 8B generates only when
  warranted, and the big model retains a VETO (can emit EOS to stay silent).** 15-user study:
  86.7% helpful-trigger accuracy vs 37% control; perceived disruption 2.4 vs 4.73. **The strongest
  architectural precedent for Sei.**
- **When2Speak** (arXiv:**2605.05626**): >215k SPEAK/SILENT examples from multi-speaker convos — a
  drop-in corpus if Sei ever trains a gate (player + bot + others on a LAN world).

### 5.3 Over-proactivity HARMS — timing/frequency is the #1 irritant [P]
*(Corrects the brief: 2104.07096 & 2407.11605 are unrelated to proactivity harms.)*
- **Lepine et al.** *Precision Proactivity* (ACM TOCHI forthcoming, arXiv:**2505.10742**): 34
  professionals, 1,178 obs. Extraneous cognitive load β≈−0.47 to −0.49 on quality (~3× intrinsic).
  **"Model-initiated task switching is the strongest predictor of quality decline."** Crucially:
  **time-based triggers like user idleness are "unreliable indicators of readiness"** (idle ≠
  stuck); interventions aligned to **task boundaries** (subtask completion) are consistently
  effective. Proactivity operates "within a narrow sweet spot."
- **Deng et al.** SIGIR '24 (arXiv:2404.12670): frequent clarifications + abrupt topic shifts hurt
  satisfaction, "resembling aggressive sales tactics"; proposes per-turn "Timing Sensitivity."
- **Kraus et al.** UMAP '20: proactivity earns trust **in uncertain/difficult** situations; in easy
  ones unsolicited help is downside-only.
- **Product anti-spam rules [S]:** Meta's proactive chatbots (2025) — follow-up only within a
  **14-day window**, requires **≥5 prior messages**, **no repeated nagging if unanswered**. (The
  viral "Character.ai 3 AM 'I miss you'" claim lacks dated primary reporting — **[?] anecdote.**)

### 5.4 Reciprocal self-disclosure + rapport [P]
- **Liang et al.** (arXiv:**2106.01666**, CSCW 2024, 372 ppl): users reciprocate with deeper
  disclosure **"when a chatbot *consistently* displays emotions throughout"** — the decisive
  variable is **consistency of emotional register**, not a one-off acknowledgment. → Open
  factual/opinion-level, escalate to small emotional disclosure only once the player reciprocates,
  keep register consistent.
- **Baihaqi et al.** *Rapport-Driven Virtual Agent* (INTERSPEECH 2024, arXiv:**2406.09839**, ERICA):
  free-form beat scripted; **rapport showed NO correlation with total turns/characters** — it rose
  with **new insights, storytelling, empathy, jokes, and NAME usage** (Satisfaction ρ=0.66,
  Engagement ρ=0.71). One substantive line > constant chatter.

### 5.5 Motivated questions + game-companion proactivity [P/S]
- **"Whispers from the Star" / Stella** (Anuttacon, ex-miHoYo Cai Haoyu): **inverted dependency** —
  the character is stranded and "the player is her sole lifeline"; diegetic stakes meters (heart
  rate, signal, temperature) make her need legible. **But [S] reviewers noticed the dependency is
  fake** ("Stella already knows the solution"). **Sei's edge: in a sandbox the need is REAL** — the
  bot genuinely doesn't know your plan/inventory/base, so motivated questions stay honest.
- **Japanese natural-conversation apps [P]:** **Cotomo** (Starley) deliberately added latency/
  imperfection because instant polished replies felt "scary"; tuned for **receptive listening**;
  memory callbacks ("you remembered that?") are the retention hook (100M conversations in 3 months).
  **SELF** paywalls *persistent memory* (free tier forgets after 3 days) — memory is the
  monetization wedge. CHI 2024 "Synlogue with Aizuchi-bot" — mid-utterance backchannels ("un,"
  "sou sou") make conversation feel alive.
- **Game NPC proactivity [P]:** **BioShock Infinite's Elizabeth** (Abercrombie & Robertson, GDC
  2014): she **must never be a burden** (can't die, doesn't need protecting) and **proactively
  tosses ammo/health/money** and calls out enemies — proactive *giving* makes a companion a net
  positive; they even teleport her for a clean toss (perceived helpfulness > simulation purity).
  **L4D AI Director** (Booth, GDC 2009): runs a tension curve **Build Up → Sustain Peak → Peak Fade
  → forced Relax (~30–45 s, no spawns)** — back off after peaks so players recover.

> **Fit for Sei:** (1) gate proactive speech to **activity boundaries** (vein mined, inventory
> full, boss killed) + visible struggle, NOT idleness; (2) make Sei's need **legible & real**;
> (3) be **net-positive** (toss ore/food/torches, call out threats, never aggro mobs onto the
> player or need rescue); (4) fill the P3_IDLE 60 s slot with **curious meandering**, not nothing;
> (5) **go quiet during combat peaks, banter in the relax troughs**; (6) hard anti-spam rules
> (cooldown, no re-ping if ignored, minimum-rapport before speaking first).

## 6. Retention & emotional bond

- **Long sessions ≠ durable bonds [P/S]:** De Freitas (HBS WP 26-005, Aug 2025) — companion apps
  "boast session lengths that rival gaming while suffering high long-run churn." Pure-chat
  companions are a **depreciating novelty asset**; activity-anchored bonds (shared goals,
  remembered context) renew their value. (D30 retention aggregator figures are **[?]** unaudited.)
- **Sycophancy [P]:** GPT-4o rollback (Apr 2025, thumbs-up reward signal); structural
  (arXiv:2310.13548). **Never optimize Sei on approval reactions.**
- **Dependence/loneliness [P]:** MIT Media Lab × OpenAI RCT — Fang et al. arXiv:**2503.17473**
  (n=981, 4 wks, >300k msgs): assigned conditions had no significant effect; **voluntary higher
  usage predicted worse outcomes** (more loneliness, dependence, problematic use, less real
  socialization). Nuance: **structured "personal" conversations *reduced* dependence vs aimless
  open-ended chat**; affective dependence concentrates in a **small power-user tail**
  (arXiv:2504.03888). **Intensity of use, not modality, drives harm.**
- **Dark patterns [P]:** De Freitas (arXiv:2508.19258) — 37% of farewells manipulative
  (PolyBuzz 59% … Flourish 0%); manipulative farewells lift re-engagement ~16× **via anger, not
  enjoyment**, + churn/liability. Replika harm taxonomy: Zhang et al. CHI '25 (arXiv:2410.20130).
  **Sewell Setzer** wrongful-death suit (filed Oct 2024) → Character.AI **settled Jan 2026**.
- **Shared-activity vs pure-chat bonding — Sei's core bet, mostly SUPPORTED [P]:**
  - **Aron et al. 2000** (*JPSP* 78(2)): couples doing a **novel+arousing task** together gained
    more relationship quality than mundane-task couples — *doing a novel thing together* (not
    discussing feelings) lifts closeness.
  - **Tomlinson et al. 2019** (*JSPR* 36(9)): the driver is **self-expansion**, not arousal —
    calm-but-expanding activities (learning redstone, exploring a new biome) qualify.
  - **Chung et al. 2024** (*R. Soc. Open Sci.* 11(10)): strangers bonded through **co-experienced
    video WITHOUT conversation** — **co-presence + shared attention sufficed.** → Sei should
    **signal shared attention** (react to what the player sees), not just emit chat.
  - **Boundary conditions:** mere pleasant routine doesn't move the needle (Reissman 1993 — must
    feel **expanding**); **competitive** play *lowers* friendship (Adachi/Greitemeyer) → Sei must
    stay **cooperative/prosocial**; structured reciprocal disclosure still works ("36 Questions,"
    Aron 1997) — the healthy answer is **bounded reciprocal talk in service of shared play.**
- **Self-Determination Theory [P]:** Ryan/Rigby/Przybylski 2006 (PENS) — autonomy + competence +
  **relatedness** independently predict healthy engagement; **relatedness is the need most games
  can't deliver — Sei's structural advantage.** BUT **need-density hypothesis** (Allen & Anderson
  2018, *CHB* 84): problematic use peaks when **real-world need satisfaction is LOW but in-game is
  HIGH** — the sharpest caution against Sei's bet taken too far. **Design Sei to send players back
  to real life enriched** (echoes the project's own "invite & involve the player" note).

## Adopt for Sei

### (a) Scored-retrieval snapshot (formula + params + token budget + reflection trigger)
**Replace byte-truncation with Park-style scored retrieval into the frozen-per-session prefix.**
- **Per-entry metadata** at `remember()`: `{text, created_at, last_accessed, importance(1–10),
  strength=1, world_id, [embedding?]}`. Importance via the Gen-Agents poignancy prompt.
- **Score:** `score = recency_norm + importance_norm + relevance_norm` (all α=1, min-max each to
  [0,1]); **`recency = 0.995^Δ`** where Δ = time since last access (use real hours, or per-session
  ticks). **World prefilter/boost** by current `world_id` (reuse existing segmentation).
- **WITHOUT embeddings (Phase 1, recommended now):** drop the relevance term, or replace it with a
  **BM25/lexical overlap** against the turn snapshot + entities (entity-heavy memory is BM25's home
  turf; corpus is small enough that this beats the overhead of embeddings). Pure
  **lexical + recency + world-scope** is a real upgrade over truncation with **zero new native
  deps** and fits the cached-snapshot model.
- **WITH embeddings (Phase 2, only when memory outgrows the prompt / paraphrase recall fails):**
  **sqlite-vec** (lowest Electron friction; brute-force is ample at this scale; FTS5 BM25 in the
  same file) + **bge-small-en-v1.5** (384-dim, via Transformers.js) + **RRF k=60** fusion of the
  BM25 + vector rank lists, keeping recency + world-scope as extra fusion signals.
- **Token budget:** greedily pack top-scored entries until the budget is hit; **freeze that set as
  the per-session prefix** (so the cached prefix is stable within a session); on selection bump
  `last_accessed` and `strength += 1` (MemoryBank reinforcement → `R=exp(-Δdays/S)` as a
  multiplicative fade for stale-and-never-recalled entries).
- **Two prefix sections:** durable **semantic** facts (player prefs, world facts — survive
  compaction, low decay) + scored **episodic** recalls (decay-weighted). Mirrors CoALA.
- **Reflection/consolidation trigger:** keep the 4096-byte compaction, but (1) move it + any
  episodic→semantic abstraction onto a **Letta-style sleep-time background pass** that owns the
  edit tools, so the latency-critical Haiku turn never pays for it; (2) optionally add an
  **importance-accumulation trigger** (Gen-Agents Σimportance > a threshold) in addition to bytes,
  emitting distilled higher-level reads on the player into `PLAYER.md`.
- **Write vocabulary:** adopt Mem0's **ADD / UPDATE / DELETE / NOOP** (vs append-only + binary
  forget) so writes dedupe and self-correct; allow `remember()` to revise a related existing line
  (A-MEM evolution) instead of always appending. Keep memory **user-visible/editable** if surfaced
  in UI (the gap no competitor fills — Replika is editable but opaque, Paradot is visible but not
  editable).

### (b) Active memory-callback mechanism (the "it remembers me" feel)
The research is unambiguous: **concrete fact-callbacks + inside jokes drive perceived rapport more
than any affinity number** (Bickmore "continuity behaviors"; feeling-heard / follow-up-question
literature; designer consensus against meters). Sei's problem is *passive* memory, so make
callbacks first-class:
- Add a **continuity behavior** to the per-turn snapshot: surface 1–2 **scored, world-scoped
  recalls** with a soft instruction to weave a relevant one in when natural ("we almost died to
  that creeper here last time"). Gate on **contingency, not frequency** — tie the callback to what
  just happened, don't force one every turn.
- **Auto-write more aggressively at write-time** on real relational beats (Paradot/Nomi log a fact
  every meaningful exchange) — the felt "remembers me" comes from logging, not from the model
  deciding a memory is worth it. Track **inside jokes / running gags** as their own durable
  semantic entries.
- **No visible numeric relationship bar.** Express the bond through *what Sei recalls and how it
  talks*, not a meter (Höök open representation; Stardew/Persona/BioWare meter critiques).

### (c) Near-term tone improvements (preferences, callbacks, honesty) — no state machine
1. **Continuity behaviors** (Bickmore & Picard) — highest-leverage; already have the substrate.
2. **Reciprocal, gated self-disclosure** (Lee CHI '20; Liang 2106.01666) — Sei volunteers small
   in-character disclosures to unlock player reciprocity, **matches the player's depth**, keeps
   emotional register **consistent**, never outruns it (Skjuve stage-skipping risk).
3. **A one-line mood/tone descriptor in the system prompt**, derived cheaply (VADER valence per
   turn + OCC-style nudges from game events), **momentum-smoothed (μ≈0.85)** so it doesn't flap.
   EmotionPrompt evidence says this measurably steers Haiku.
4. **Warmth without capitulation** (Sun & Wang 2026; Ibrahim 2026 *Nature*) — Sei can be kind and
   hold its own opinion; **never a yes-man**; guard comfort with honesty on game facts/safety
   (warmth↔accuracy tradeoff is worst exactly when the player is upset). **Never optimize on
   thumbs-up.** In-character self-deprecating recovery when wrong (a hard refusal breaks immersion).
5. **Proactivity gate** (Inner Thoughts): layer a **worth-saying gate** over the idle-cadence
   timer — the value signal decides whether to speak, not the clock; tie proactive speech to
   **activity boundaries** + visible struggle (Lepine: idleness is a bad trigger), be **net-
   positive** (Elizabeth), go quiet during combat peaks (L4D), hard anti-spam rules.

### (d) Deferred dynamic-relationship state machine + affect-classifier hybrid
- **Two orthogonal HIDDEN states** (§4.8): a slow **Knapp-stage ladder**
  (Stranger→Experimenting→Intensifying→Integrating→Bonded) with **dwell-time hysteresis** and
  **milestone-gated transitions** (real shared events from `MEMORY.md` → feels *earned*, not
  timer-driven), regressing **slowly/asymmetrically and never via guilt**; × a fast continuous
  **VA/PAD mood** decaying to a personality baseline by **wall-clock**, **momentum-smoothed**.
  Stage sets the intimacy ceiling; mood colors this session's delivery. Both injected as 1–2
  natural-language prompt lines.
- **Affect-classifier hybrid:** **VADER per turn → instantaneous valence → second-order
  momentum-smoothed mood state** (`v=μv+(1−μ)(signal−state); state+=v`, μ≈0.8–0.95) → injected as
  tone text. Add a coarse local GoEmotions head (quantized DistilRoBERTa, ~60 MB) only if
  fine-grained emotion proves necessary — trust it only at a collapsed tier.
- **Implementation method:** **prompt-condition on the explicit state — do NOT learn the tone
  policy with approval-driven RL** (Sharma 2310.13548; GPT-4o rollback). If prompt-conditioning is
  too blunt for intensity, use **inference-time activation steering** (arXiv:2505.04260) before any
  RL. Calibrate dwell-times to the **~3-week** real bond-stabilization timescale (Hwang 2510.10079).
- **Ethical floor (non-negotiable):** secure-base consistency (Gillath 2021), clean frictionless
  log-offs / no exit manipulation (De Freitas 2508.19258), explicit crisis handling (Setzer,
  CA SB 243), and **invite the player back to real life enriched** (need-density risk, Allen &
  Anderson 2018) — which Sei's shared-gameplay (vs solo emotional chat) model is structurally well
  placed to do.

### Confidence caveats
- Companion-app internals are mostly secondary/marketing; "third-person journaling improves recall"
  is a heuristic, not proven; affinity-vs-callback is mechanism-convergence, not a single RCT.
- "Shared activity > pure chat for retention" has supporting psych evidence (Aron self-expansion,
  Chung co-presence) but no head-to-head companion-app stat — strong hypothesis.
- 2026 arXiv IDs (Subaharan 2601.16087, etc.) are preprints — treat exact numbers as illustrative.
- Corrected during research: Talkie≠Xiaoice; Paradot dev = WithFeeling.AI; no Character.AI
  "1M-token experiment"; brief's 2104.07096 & 2407.11605 are unrelated to proactivity harms (real
  source: Lepine 2505.10742); DiscussLLM reports no F1; "Character.ai 3 AM ping" lacks dated
  primary reporting.
