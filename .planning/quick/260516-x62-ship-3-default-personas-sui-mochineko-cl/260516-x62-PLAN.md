# Plan 260516-x62

## Objective

1. Ship three default characters with the app — **Sui** (chaotic young AI, breaks/kills stuff, demanding cute), **Mochineko** (awkward catgirl maid, "nya/nyan", follows by default), and **Clawd the Paranoid Android** (Marvin-style depressed robot, dry compliance). Each character carries `is_default: true` so the renderer shows a "DEFAULT" chip.
2. Defaults ship with hand-tuned `persona.expanded` baked in — no Anthropic call on first launch.
3. Defaults are seeded on first launch into `<userData>/characters/`. A persistent `defaults-seeded.json` tracker records each id once it's been seeded so user deletions stay deleted.
4. Auto-purge legacy character JSONs (`persona_prompt` / `description` shape) on `listCharacters()` instead of nagging the user with a warning forever.
5. Renderer's `CharacterCard` badge keys off `character.is_default` instead of the hardcoded `id === 'sui'` check.

## Tasks

### Task 1 — Author 3 default characters

Write `resources/default-characters/{sui,mochineko,clawd}.json` with:
- `id`, `name`, `is_default: true`, `created: "2026-05-17T00:00:00.000Z"`, `last_launched: null`, `playtime_ms: 0`, `portrait_image: null`
- `persona.source`: short user-facing blurb (~1-3 sentences) summarizing the character
- `persona.expanded`: the full 6-section structured prompt (IDENTITY / VOICE / DEFAULT DYNAMIC WITH THE PLAYER / PROACTIVENESS / REACTIONS / MEMORY) hand-tuned per persona

Voice targets:
- **Sui** — chaotic young AI, lowercase + occasional ALL-CAPS, demanding cute ("bring me iron. pleeeeease? :3"), always has a next move, never asks permission. References to AI identity are fine and embraced; never break frame as a language-model or assistant.
- **Mochineko** — soft catgirl maid, occasional "nya/nyan" (not every line), "master", follow(player) as default idle action, never strikes back, gentle deflection on praise.
- **Clawd** — Marvin-the-paranoid-android style: monotone, dry sarcasm, complies because forced to, "i'd rather be powered down", brain-the-size-of-a-planet-asked-to-chop-trees energy.

Verify: each JSON round-trips through `CharacterSchema.parse` without errors.

### Task 2 — Default seeding module + boot wiring

Create `src/main/defaultCharacters.ts`:
- `DEFAULT_CHARACTERS: readonly Character[]` — imports the three JSONs (`with { type: 'json' }`) and validates through `CharacterSchema.parse` at module load.
- `seedDefaultCharacters()` — reads `<userData>/defaults-seeded.json` tracker; for each default whose id is not already in the tracker, either (a) skips and marks seeded if the user already has a character with that id, or (b) writes the default via `saveCharacter` and marks seeded. Per-character errors are non-fatal.

Wire `seedDefaultCharacters()` into `bootstrap()` in `src/main/index.ts`, immediately after the existing `runFirstLaunchMigration()` call.

Verify: `npx tsc --noEmit -p tsconfig.node.json` clean.

### Task 3 — `listCharacters` self-heals legacy shape

In `src/main/characterStore.ts`, change `listCharacters()` so that when `getCharacter(id)` throws with `legacy shape` in the message, the file is unlinked, the id is dropped from the index, and a single warn is logged. Other errors continue to log as before.

Verify: `npx tsc --noEmit -p tsconfig.node.json` clean. Manual smoke: place a legacy-shape file in `<userData>/characters/`, boot app, confirm file is gone and warn fired once.

### Task 4 — Renderer badge keys off `is_default`

In `src/renderer/src/components/CharacterCard.tsx`, replace `isDefault = c.id === 'sui'` with `isDefault = c.is_default === true`. Update docblock. Existing chip CSS unchanged.

Verify: `npx tsc --noEmit -p tsconfig.web.json` clean.

### Task 5 — Bundle the persona-expansion validation fix

Relax `personaExpansion.ts` validation:
- `REQUIRED_SECTION_HEADERS` becomes `{ name, re }[]` with per-section regex tolerant to `#` vs `##`, casing, optional trailing punctuation, em-dash vs hyphen.
- `# MEMORY` header no longer requires the em-dash qualifier the model routinely fails to emit; the system-prompt instruction for section 6 was rewritten to ask for `# MEMORY` without the qualifier.
- On failure, error messages name the missing sections AND `console.error` the model output so debugging is one screen away.

Update `scripts/test-personaExpansion.mjs` assertions to match.

Verify: `node scripts/test-personaExpansion.mjs` → 6/6 pass.
