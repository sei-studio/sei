# Quick 260526-00d — Cluster J: NCMEC CyberTipline Reporting Scaffold

## Scope

18 USC §2258A imposes mandatory NCMEC reporting on electronic communication
service providers when they obtain actual knowledge of apparent CSAM. Fines:
$150K first knowing-and-willful failure, $300K each subsequent. The COPPA
DOB age gate (Cluster I Task 1) prevents under-13 signups; this cluster
ships the detection-side audit trail and queue so Sei can truthfully attest
to NCMEC reporting before going live to users.

## Design — Queue First, Submit Later

End-to-end queue wiring (in-request, automated):
1. `moderate-character-images` flags an image as CSAM via SightEngine.
2. The function persists the verdict to `characters.moderation_status` AND
   queues a row into `public.ncmec_reports_sent` with `ncmec_submitted_at = NULL`.
3. The user-facing response is unchanged — flagged users still see "flagged."

Submission side (operator-managed, asynchronous):
1. Operator runs `scripts/admin-submit-ncmec.ts` (cron or ad-hoc).
2. CLI drains unsent rows, POSTs each to NCMEC's CyberTipline ISP web
   service, stamps `ncmec_submitted_at`/`ncmec_response_status_code`/`ncmec_response_body`.

Rationale for queue-first vs. synchronous NCMEC POST: NCMEC's API has 1-2
week ESP-registration lead time and no published uptime SLA. Blocking the
user-facing moderation request on NCMEC reachability would degrade the
moderation UX during NCMEC outages. The queue row IS the record of
knowledge of CSAM (satisfies 18 USC §2258A(a)(2)); the actual submission
follows asynchronously "as soon as reasonably possible" via the CLI.

## USER-ACTIONS (operator runbook before going live)

1. **Register Sei as an Electronic Service Provider with NCMEC.**
   - Submit ESP registration at https://report.cybertip.org/ispws/documents.
   - Processing time: 1-2 weeks. Without this, NCMEC POSTs will be rejected.
   - This is the LONGEST lead-time item — start ASAP.

2. **Set Edge Function secret for the queue endpoint:**
   ```bash
   supabase secrets set NCMEC_ADMIN_TOKEN=$(openssl rand -hex 32)
   ```
   Save the generated token in your operator password vault. Required for any
   ad-hoc curl invocations of `submit-ncmec-report`; NOT used by the
   moderate-character-images auto-queue path (that path uses service_role
   directly via `adminClient`).

3. **Set Supabase Dashboard env vars for the operator CLI:**
   ```bash
   # On the operator workstation (.env or environment):
   export SUPABASE_URL=https://<project-ref>.supabase.co
   export SUPABASE_SERVICE_ROLE_KEY=...
   export NCMEC_ESP_USERNAME=...  # from NCMEC ESP registration
   export NCMEC_ESP_PASSWORD=...  # from NCMEC ESP registration
   ```

4. **Verify the NCMEC API endpoint + payload schema.**
   - `https://report.cybertip.org/ispws/wsdl` is SOAP/WSDL.
   - The JSON-over-HTTPS variant may or may not exist at the same base URL.
   - Run a single submission via `npx tsx scripts/admin-submit-ncmec.ts --limit=1` against a test row and inspect the NCMEC response. If 404/415: web-form fallback at https://report.cybertip.org/ until the JSON path is confirmed, OR adapt the CLI to emit SOAP XML.

5. **Apply the migration + deploy the Edge Function:**
   ```bash
   supabase db push  # applies 20260528001700_ncmec_reports.sql
   supabase functions deploy submit-ncmec-report
   supabase functions deploy moderate-character-images  # re-deploy with queue wiring
   ```

6. **Verify the queue wiring end-to-end on staging** before flipping production:
   - Manually flag a test character (or use a known-flagged image fixture).
   - Confirm a row lands in `ncmec_reports_sent` with `ncmec_submitted_at IS NULL`.
   - Run `npx tsx scripts/admin-submit-ncmec.ts --dry-run` — should print the row.
   - Run `npx tsx scripts/admin-submit-ncmec.ts --limit=1` against staging — should POST to NCMEC and stamp the row (assuming NCMEC accepts staging traffic).

7. **Schedule the CLI on operator cron** (e.g., every 4 hours):
   ```cron
   0 */4 * * * cd /path/to/sei && env $(cat .env | xargs) npx tsx scripts/admin-submit-ncmec.ts >> /var/log/sei-ncmec.log 2>&1
   ```
   Or run manually until automated submission is verified. 18 USC §2258A(a)(2)
   "as soon as reasonably possible" — a 4-hour cron easily meets that bar.
