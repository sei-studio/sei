<objective>
Cluster J — NCMEC CyberTipline reporting scaffold. 18 USC §2258A imposes mandatory NCMEC reporting on
electronic communication service providers when they obtain actual knowledge of apparent CSAM. This
quick-task wires the QUEUE side end-to-end (DB table + Edge Function wrapper + moderate-character-images
trigger + operator CLI). The actual NCMEC API submission is operator-managed (1-2 week ESP registration
required before the API is even callable) — the scaffold lets us record evidence the instant SightEngine
flags CSAM so we are not at the mercy of NCMEC API availability for legal compliance.

Purpose: Be able to truthfully attest to NCMEC reporting (18 USC §2258A) before going live to users.
Output: Migration + Edge Function + moderate-character-images patch + operator CLI; SAFER queue-first
design where actual NCMEC POST failures never block the user-facing moderation response.
</objective>

           operator_notes: body.operator_notes ?? null }`.
      - Return 202 `{ status: 'queued', ncmec_id: <inserted row uuid> }` on success.
      - Return 500 `{ error: 'db_insert_failed', detail }` on insert failure.

  <action>
    Create a Node CLI mirroring `scripts/admin-resolve-report.ts` shape (no dotenv, env from process.env,
    same exit-code vocabulary).

    Usage: `npx tsx scripts/admin-submit-ncmec.ts [--dry-run] [--limit=N]`
      - default limit = 50.
      - `--dry-run` prints the rows without POSTing to NCMEC.

    Env:
      - SUPABASE_URL                 (required)
      - SUPABASE_SERVICE_ROLE_KEY    (required)
      - NCMEC_ESP_USERNAME           (required unless --dry-run)
      - NCMEC_ESP_PASSWORD           (required unless --dry-run)
      - NCMEC_API_URL                (optional — defaults to `https://report.cybertip.org/ispws/`)

    Behavior:
      1. SELECT * from ncmec_reports_sent where ncmec_submitted_at is null order by created_at asc limit N.
      2. For each row:
         - If --dry-run: console.log({ id, character_id, owner_id, created_at }) and continue.
         - Build the CyberTipline payload as a placeholder JSON envelope:
             {
               espName: 'Sei',
               incidentSummary: 'AI-generated character portrait flagged by SightEngine nudity-2.1 + face-attributes minor signal',
               incidentDateTime: row.created_at,
               additionalInfo: row.operator_notes,
               contentURL: row.raw_evidence_url,
               reporterContact: { internalUserId: row.reporter_internal },
               suspectInfo: { internalUserId: row.owner_id },
               providerResponse: row.raw_provider_response,
             }
         - POST to NCMEC_API_URL with `Authorization: Basic <base64(user:pass)>` and 30s timeout
           (AbortController). NCMEC's real ISP web service is SOAP/WSDL — this scaffold POSTs JSON.
           Doc comment: operator MUST verify the actual NCMEC API endpoint + payload schema (SOAP vs JSON
           varies; current `report.cybertip.org/ispws/` redirects to a WSDL; the JSON-over-HTTPS variant may
           or may not exist). If the JSON POST fails with 404 / 415 / etc., the operator falls back to the
           CyberTipline web form for now.
         - On non-2xx OR network error: log `failed: <id> <status> <body>` and continue (best-effort, no
           exit-non-zero — one bad row should not block the rest of the batch).
         - On 2xx: UPDATE ncmec_reports_sent set ncmec_submitted_at=now(), ncmec_response_status_code=<status>,
           ncmec_response_body=<text> where id=<row.id>. Log `submitted: <id>`.
      3. After the loop: print `done. submitted=<S> failed=<F> dry_run=<bool>`.

    NEVER log credentials. Auth header is built locally inside the POST scope only.
    `.env` is gitignored — same loader expectation as admin-resolve-report.ts (operator sources env or
    uses `env $(cat .env | xargs) npx tsx ...`).

    Exit codes:
      0 — success (even with partial failures — those are logged, not fatal).
      1 — usage / arg error.
      2 — env missing.
      3 — DB error during the initial SELECT.

    Comment at top of file must reference 18 USC §2258A and the queue-first design from Task 1.
  </action>
