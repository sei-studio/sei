<objective>
Build the Tier-2 runtime mechanism first: hand-validate a target expanded-persona
format on one concrete character, then wire the runtime so a compressed persona
core (`# CORE`) is lifted from `persona.expanded` and re-injected at the recency
tail of every turn. This is the central PERSONA-03 anti-drift mechanism. Also add
the honest-mistakes instruction (PERSONA-06) to the universal baseline. Per the
user's workflow guidance, the runtime assembly and a proven hand-authored target
come BEFORE the expander generalization (Plan 02), because the expander's only job
is to reproduce a format already shown to work at runtime.

Purpose: Persona drift is attention decay; re-injecting a compressed core at high
recency is the retraining-free fix validated in research. This plan proves the
mechanism on a concrete example so the expander can be generalized against it.

Output: A `renderCore()` extractor, a recency-tail core block in
`composeSeedBlocks`, a honest-mistakes baseline instruction, and a hand-authored
target `# CORE` / `# VOICE` example used to validate drift resistance.
</objective>
