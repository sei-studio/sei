## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User drag-drop → renderer | Browser DataTransfer.files exposes raw File objects; we validate type, magic, dimensions client-side before hitting IPC |
| Renderer → main (`skin:apply`) | base64 PNG bytes cross the IPC boundary; main re-validates everything inside `applyPng` per Plan 02 |
| Renderer → external `<img src=skinServerBaseUrl/...>` | The 3D preview canvas / 2D fallback fetch from the loopback skin server; same-origin policy allows it (CSP unaffected) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-T7 | T (Tampering) | drag-drop file path | mitigate | Client-side magic-byte + IHDR dimension validation rejects non-PNG / non-64x64 before sending bytes to main. Main re-validates (defense-in-depth). |
| T-09-X1 | XSS-adjacent | base64 PNG injected into data: URL | accept | data: URLs in `<img src>` cannot execute JS regardless of payload. The base64 string is computed in-renderer from user input and consumed by `<img>` / `viewer.loadSkin()` — both image-only sinks. |
| T-09-D4 | D (Denial of Service) | Mojang search rate-limit cascade | mitigate | UI-SPEC error copy surfaces "rate-limiting" → user waits a minute. Plan 03's 15s timeout caps each call. |
| T-09-I4 | I (Information Disclosure) | data: URL retains base64 in DOM | accept | Same-origin; the URL never leaves the renderer process. |
