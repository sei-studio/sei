<objective>
Add the server-authoritative per-hour vision cap (VIS-07 / D-09) to the Fly.io proxy. Explicit `visualize`-triggered LLM turns are LLM-driven and therefore UNBOUNDED (the model could loop `visualize`), so the cap is the hard ceiling independent of credit balance. Idle renders are already bounded by the idle-tick cadence and MUST NOT route through this gate.

Purpose: This plan is fully independent of the render path (the proxy is a separate subsystem on Fly.io) — it runs in Wave 1 in parallel with the de-risk spike. It establishes the `/vision/v1/messages` path the bot routes explicit-vision turns to (the bot-side routing is wired in 15-06).

Output: a `vision_hourly` bucket kind (TS union + a migration extending the `rate_buckets` CHECK constraint), a `visionHourlyGate` middleware mirroring `personaDailyGate`, and a `POST /vision/v1/messages` route chained `originLockGate -> ipRateLimitGate -> verifyJwt -> visionHourlyGate -> forwardToAnthropic`.
</objective>
