<objective>
Surface the auto-render opt-in (VIS-04 UI half) and its cost communication (D-05/D-06/D-07) in the renderer — fully independent of the bot render path, so it runs in Wave 2. ONE toggle line in Settings; turning it ON opens a confirm popup that plainly says auto-look "uses more playtime" with NO numbers (D-06/PROXY-05); confirming writes `vision_auto_render` through to UserConfig (the field added in 15-03). The toggle is DISABLED for non-VLM providers via the `visionCapable` signal surfaced in 15-03 (D-10/VIS-03). The cost surfaces where users already look: when auto-render is ON, the Playtime/Credits "~Xh" estimate shrinks via a vision multiplier (D-07).

Purpose: This honors the user decision that cost communication is a shrunk playtime figure + a plain-language popup, NOT a scary number at toggle time (PROXY-05 — no token counts anywhere). Single global toggle scopes to the active character in practice (single-bot-at-a-time, D-05). The non-VLM disable is now a delivered behavior (not a noted limitation) because 15-03 surfaces `capabilities.vision` to the renderer.

Output: `VisionAutoRenderConfirmModal.tsx` (cloned from SwitchBackendConfirmModal, reusing the shared modal CSS); a toggle row + confirm-trigger in SettingsScreen wired `disabled` to `useUiStore.visionCapable`; a `VISION_MULTIPLIER` in playtimeEstimate.ts applied when auto-render is ON.
</objective>
