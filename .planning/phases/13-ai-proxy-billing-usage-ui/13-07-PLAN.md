TypeScript test cases:
1. First request (no row exists) → rate_buckets INSERTS with count=1, returns {allowed: true}.
2. Within window, count below limit → UPDATE count=count+1, returns {allowed: true}.
3. Within window, count equals limit (e.g., 20th call in 60s for trial RPM) → 21st INSERT/UPDATE would set count=21 > 20; RPC rolls back via the UPDATE-decrement clause; returns {allowed: false, retry_after_seconds: <int>}.
4. After window expired (window_start < now - 60s) → UPDATE resets count to increment and window_start to now → returns {allowed: true}.
5. checkAllBuckets fails fast: if RPM check returns {allowed: false}, ITPM and daily_dollar checks are NOT called (no wasted DB roundtrip).
6. Plan tier cache: two consecutive checkAllBuckets calls within 30s → second one does NOT re-read subscription_status (mock counts only one .from('subscription_status').select call).
7. Tier resolution: subscription_status.status='active' → subscriber caps; status='cancelled' → trial caps (cancelled but balance remains); no row → trial caps.
8. ITPM with cache-read exclusion: input_tokens=50_000 and cache_read=200_000 → only 50_000 counts toward ITPM (RESEARCH note: matches Anthropic Tier 2 accounting). For v1.0, estimateInputTokens (13-05) already excludes the cached system prefix from its count — confirm via the tokenize.ts comment.
9. retry_after_seconds for RPM bucket equals 60 - elapsed_seconds when limit hit mid-window; for daily_dollar equals seconds_until_window_end.

`supabase/migrations/20260524000100_rate_buckets_rpc.sql` (per RESEARCH §Pattern 6, with security definer + execute grant):

```sql
create or replace function public.check_and_increment_bucket(
  p_user_id uuid,
  p_bucket_kind text,
  p_increment bigint,
  p_limit bigint,
  p_window_seconds int
) returns table(allowed boolean, retry_after_seconds int)
language plpgsql security definer set search_path = public as $$
declare
  v_current_count bigint;
  v_window_start timestamptz;
  v_now timestamptz := now();
begin
  insert into public.rate_buckets (user_id, bucket_kind, count, window_start)
  values (p_user_id, p_bucket_kind, p_increment, v_now)
  on conflict (user_id, bucket_kind) do update
    set
      count = case
        when rate_buckets.window_start + (p_window_seconds || ' seconds')::interval < v_now
          then EXCLUDED.count
        else rate_buckets.count + EXCLUDED.count
      end,
      window_start = case
        when rate_buckets.window_start + (p_window_seconds || ' seconds')::interval < v_now
          then v_now
        else rate_buckets.window_start
      end
    returning count, window_start into v_current_count, v_window_start;

  if v_current_count > p_limit then
    -- Roll back the increment so the user can retry without "burning" a slot
    update public.rate_buckets
      set count = count - p_increment
      where user_id = p_user_id and bucket_kind = p_bucket_kind;
    return query select false,
      greatest(1, ceil(extract(epoch from (v_window_start + (p_window_seconds || ' seconds')::interval - v_now)))::int);
  else
    return query select true, 0;
  end if;
end;
$$;
grant execute on function public.check_and_increment_bucket(uuid, text, bigint, bigint, int) to service_role;
```

`proxy/src/rateLimit/buckets.ts`:
```ts
import { getAdminClient } from '../supabase.js';

export type BucketKind = 'rpm' | 'itpm' | 'otpm' | 'daily_dollar';
export type LimitResult = { allowed: true } | { allowed: false; kind: BucketKind; retry_after_seconds: number };

type Tier = 'trial' | 'subscriber';
type Caps = { rpm: bigint; itpm: bigint; daily_dollar: bigint };

const TRIAL_CAPS: Caps = { rpm: 20n, itpm: 30_000n, daily_dollar: 5_000_000n };
const SUB_CAPS: Caps = { rpm: 60n, itpm: 200_000n, daily_dollar: 20_000_000n };

// In-memory tier cache to skip subscription_status read on hot path
const tierCache = new Map<string, { tier: Tier; expiresAt: number }>();
const TIER_CACHE_TTL_MS = 30_000;

export async function getUserTier(userId: string): Promise<Tier> {
  const cached = tierCache.get(userId);
  if (cached && cached.expiresAt > Date.now()) return cached.tier;
  const admin = getAdminClient();
  const { data } = await admin.from('subscription_status').select('status').eq('user_id', userId).maybeSingle();
  const tier: Tier = data?.status === 'active' ? 'subscriber' : 'trial';
  tierCache.set(userId, { tier, expiresAt: Date.now() + TIER_CACHE_TTL_MS });
  return tier;
}

async function checkBucket(
  userId: string,
  kind: BucketKind,
  increment: bigint,
  limit: bigint,
  windowSeconds: number,
): Promise<LimitResult> {
  const admin = getAdminClient();
  const { data, error } = await admin.rpc('check_and_increment_bucket', {
    p_user_id: userId,
    p_bucket_kind: kind,
    p_increment: increment.toString(),
    p_limit: limit.toString(),
    p_window_seconds: windowSeconds,
  });
  if (error) throw error;
  const row = (data as Array<{ allowed: boolean; retry_after_seconds: number }>)[0];
  if (!row?.allowed) return { allowed: false, kind, retry_after_seconds: row?.retry_after_seconds ?? 60 };
  return { allowed: true };
}

export async function checkAllBuckets(
  userId: string,
  estInputTokens: number,
  reservationMicro: bigint,
): Promise<LimitResult> {
  const tier = await getUserTier(userId);
  const caps = tier === 'subscriber' ? SUB_CAPS : TRIAL_CAPS;

  // 1. RPM
  const rpm = await checkBucket(userId, 'rpm', 1n, caps.rpm, 60);
  if (!rpm.allowed) return rpm;
  // 2. ITPM — cache-read tokens already excluded by estimateInputTokens (13-05)
  const itpm = await checkBucket(userId, 'itpm', BigInt(estInputTokens), caps.itpm, 60);
  if (!itpm.allowed) return itpm;
  // 3. daily_dollar — reservation pre-deducted, not actual; rolls back to actual after settle
  const daily = await checkBucket(userId, 'daily_dollar', reservationMicro, caps.daily_dollar, 86_400);
  if (!daily.allowed) return daily;

  return { allowed: true };
}
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-07-01 | Tampering | Concurrent requests racing on bucket count | mitigate | Postgres UPSERT on PK (user_id, bucket_kind) serializes via row-level locks; UPDATE-decrement on over-limit is in the same transaction |
| T-13-07-02 | Tampering | Stale tier cache lets cancelled subscriber keep subscriber caps | accept | 30s TTL bounds staleness; user's next request post-cancellation will downgrade. Acceptable for v1.0 — alternative is per-request DB read |
| T-13-07-03 | DoS | Single user exhausting Anthropic Tier 2 org cap | mitigate | Per-user RPM (≤60) × concurrent subscriber count must stay under 1000 (Tier 2 org). Subscriber cap 60 RPM means ~16 simultaneous subscribers before risk. Alert ops on sustained Anthropic 429s (13-23) |
| T-13-07-04 | Information Disclosure | RPC error leaks DB | mitigate | Wrapper re-throws; caller in 13-10 emits sentinel `internal_error` |
| T-13-07-05 | Tampering | retry_after_seconds clock skew between proxy and DB | accept | DB clock authoritative; proxy returns the value verbatim; small client-side skew is non-security |
| T-13-07-06 | Tampering | Hostile client forging headers to spoof higher tier | n/a | Tier is read server-side from subscription_status; never client-provided |
