Test cases:
1. settle(reservationId, usage={input:1000, cache_creation:0, cache_read:0, output:100}, 'claude-haiku-4-5', 'msg-abc') → RPC called with p_actual_micro=BigInt(1500).toString(), p_anthropic_call_id='msg-abc'. Returns void.
2. Cache-hit case: settle with usage={input:0, cache_creation:0, cache_read:50000, output:100} → actual=5000+500=5500 µ$. If reservation was 1.25M µ$ (worst case), settle UPDATEs micro=5500. Refund delta = 1.25M - 5500 = 1.24M µ$ — observable in the next ledger_balance read.
3. settleAsRefunded(reservationId) → calls settle_consumption with p_actual_micro=0 — UPDATE clamps to 0 (LEAST). Then a subsequent SQL not in scope here flips reservation_state to 'refunded'. Actually simpler: settleAsRefunded UPDATEs directly via supabase-js `update().eq()` with reservation_state='refunded', micro=0. (Plan owner: choose path; recommendation: extend the SQL RPC to accept an optional refund flag, but for v1.0 just `from('ledger_consumption').update({reservation_state:'refunded', micro:0}).eq('id', resId).eq('reservation_state', 'reserved')` is simpler and equally safe.)
4. Idempotent re-settle: the WHERE reservation_state='reserved' clause in the RPC makes settle() a no-op if called twice (RPC affects 0 rows on the second call). Test asserts no error is thrown.
5. RPC error → settle rejects; caller (forward.ts) logs but doesn't fail the user-facing response (we've already streamed bytes).
6. usage with all zeros (defensive default when Anthropic stream truncates) → actual=0, settle UPDATEs micro to LEAST(reservation, 0) = 0, effectively a full refund. Test asserts this is the chosen behavior — and that 13-08 forward.ts has a TODO to log "stream interrupted, refunding" in this branch.

   IMPORTANT: PATTERNS pitfall #5 says when stream closes without message_delta, settle at FULL reservation amount (no refund). v1.0 chooses the SAFER variant: log + settle at reservation. This is implemented in 13-08 forward.ts, NOT settle.ts — settle.ts honors whatever actualMicro the caller passes.

```ts
import { getAdminClient } from '../supabase.js';
import { computeMicroDollarCost, type Usage } from '../anthropic/pricing.js';

export async function settle(
  reservationId: string,
  usage: Usage,
  model: string,
  anthropicCallId: string | null,
): Promise<void> {
  const actualMicro = computeMicroDollarCost(usage, model);
  const admin = getAdminClient();
  const { error } = await admin.rpc('settle_consumption', {
    p_reservation_id: reservationId,
    p_actual_micro: actualMicro.toString(),
    p_anthropic_call_id: anthropicCallId,
  });
  if (error) throw error;
}

export async function settleAsRefunded(reservationId: string): Promise<void> {
  // Used when upstream returned 429 (D-44) or stream aborted without usage.
  // Direct UPDATE — the RPC doesn't model a 'refunded' state mutation, so we go through the table.
  // The WHERE reservation_state='reserved' clause makes it idempotent.
  const admin = getAdminClient();
  const { error } = await admin
    .from('ledger_consumption')
    .update({ reservation_state: 'refunded', micro: 0 })
    .eq('id', reservationId)
    .eq('reservation_state', 'reserved');
  if (error) throw error;
}

export async function settleAtReservation(reservationId: string, anthropicCallId: string | null): Promise<void> {
  // Used when the stream closes WITHOUT a final usage event (PATTERNS pitfall #5).
  // Marks as settled at the original reservation amount — no refund. Safer default.
  const admin = getAdminClient();
  const { error } = await admin
    .from('ledger_consumption')
    .update({ reservation_state: 'settled', anthropic_call_id: anthropicCallId })
    .eq('id', reservationId)
    .eq('reservation_state', 'reserved');
  if (error) throw error;
}
```
