  <behavior>
Cases (each is a separate vitest test):
1. Missing Authorization header → returns 401 with body `{ error: 'missing_jwt' }`, `c.var.userId` never set, next() never called.
2. Authorization header without "Bearer " prefix → 401 missing_jwt.
3. Expired JWT (jose throws code === 'ERR_JWT_EXPIRED') → 401 `{ error: 'expired_jwt' }`.
4. JWT with invalid signature (jose throws JWSSignatureVerificationFailed) → 401 `{ error: 'invalid_jwt', detail: <msg> }`.
5. JWT with wrong audience (jose throws because audience mismatches) → 401 invalid_jwt.
6. JWT verified but payload.sub is undefined or not a string → 401 `{ error: 'invalid_jwt', detail: 'no sub claim' }`.
7. Happy path: valid JWT with sub='abc-uuid' and aud='authenticated' → sets c.var.userId='abc-uuid', calls next(), next handler sees userId.

Tests mock `jose.jwtVerify` (do NOT spin up a real JWKS server). Pattern:

`proxy/src/auth/verifyJwt.ts`:
```ts
import type { MiddlewareHandler } from 'hono';
import { jwtVerify } from 'jose';
import { JWKS } from './jwks.js';

export type AuthVars = { userId: string };

export const verifyJwt: MiddlewareHandler<{ Variables: AuthVars }> = async (c, next) => {
  const auth = c.req.header('Authorization');
  if (!auth || !auth.startsWith('Bearer ')) {
    return c.json({ error: 'missing_jwt' }, 401);
  }
  const token = auth.slice('Bearer '.length);

  try {
    const { payload } = await jwtVerify(token, JWKS, { audience: 'authenticated' });
    if (typeof payload.sub !== 'string' || payload.sub.length === 0) {
      return c.json({ error: 'invalid_jwt', detail: 'no sub claim' }, 401);
    }
    c.set('userId', payload.sub);
    await next();
    return;
  } catch (err: unknown) {
    const code = (err as { code?: string })?.code;
    if (code === 'ERR_JWT_EXPIRED') {
      return c.json({ error: 'expired_jwt' }, 401);
    }
    const msg = err instanceof Error ? err.message : 'verification failed';
    return c.json({ error: 'invalid_jwt', detail: msg }, 401);
  }
};
```

PATTERNS pitfall: do NOT leak inner error stack — only `err.message` (one line) goes back to the client. Production logging happens via a Sentry hook in plan 13-10.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-04-01 | Spoofing | Forged JWT with no signature | mitigate | jose.jwtVerify checks signature against JWKS-fetched public key; signature failures raise JWSSignatureVerificationFailed → 401 invalid_jwt |
| T-13-04-02 | Spoofing | JWT with valid signature but wrong audience | mitigate | jwtVerify(token, JWKS, { audience: 'authenticated' }) — audience mismatch throws |
| T-13-04-03 | Replay | Expired JWT replay | mitigate | jose checks exp automatically; ERR_JWT_EXPIRED → 401 expired_jwt — client must refresh |
| T-13-04-04 | Information Disclosure | Detail field leaks JOSE internal error string | mitigate | Detail truncated to err.message only (single line, no stack); production logger filters at sentinel |
| T-13-04-05 | DoS | JWKS endpoint flooding | mitigate | 1h cacheMaxAge + 30s cooldownDuration in jose's createRemoteJWKSet; only one network round-trip per proxy boot |
| T-13-04-06 | Tampering | JWT with sub=null or sub=number | mitigate | Explicit `typeof payload.sub !== 'string' || payload.sub.length === 0` guard before c.set; invalid sub → 401 invalid_jwt |
