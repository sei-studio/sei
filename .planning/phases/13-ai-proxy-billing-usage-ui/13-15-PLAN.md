function buildSdkOptions(config) {
  if (config.anthropic.cloudMode) {
    return {
      baseURL: config.anthropic.cloudMode.baseURL,           // e.g. https://api.sei.gg
      authToken: process.env.CLOUD_PROXY_JWT ?? config.anthropic.cloudMode.authToken,
      apiKey: 'unused-but-required-by-sdk',
    };
  }
  return { apiKey: config.anthropic.api_key };
}

const sdk = new Anthropic(buildSdkOptions(config));
```

CRITICAL: read process.env.CLOUD_PROXY_JWT on EVERY SDK construction (which happens per bot session). For longer-lived sessions, the JWT rotation pump (13-14) updates process.env.CLOUD_PROXY_JWT in-place; the next request the SDK initiates will pick up the new value because `authToken` is read by the SDK per request (verify by inspecting the @anthropic-ai/sdk source: it reads `this.authToken` at request time, not at construction time).

If the SDK does NOT re-read authToken per request (i.e. it caches at construction), this plan needs a follow-up to either:
  (a) recreate the SDK instance when CLOUD_PROXY_JWT changes (bot listens for cloud-jwt-update and rebuilds the sdk), OR
  (b) replace SDK with a thin fetch-based client for cloud-proxy mode.

PATTERNS warning: this is the trickiest part of sub-delivery (a). Verify SDK behavior via a quick test or by reading node_modules/@anthropic-ai/sdk/dist/core.js for the auth handling. If the SDK caches, add a `parentPort.on('message', msg => { if (msg.kind === 'cloud-jwt-update') sdk = new Anthropic(buildSdkOptions(config)) })` re-init pattern inside this file.

Per PATTERNS Pitfall: "JWT rotation: a Sei session JWT typically expires in 1h. The bot process can run for many hours. proxyJwtFetcher.ts must refresh the JWT before expiry and the bot must accept new tokens mid-session — easiest is a periodic `sdk.authToken = newJwt` reassignment (the SDK reads the field per request)."

For safety in v1.0: implement BOTH approaches:
  1. process.env.CLOUD_PROXY_JWT read per-call (if SDK supports)
  2. Listen for cloud-jwt-update message and either set `sdk.authToken = msg.jwt` OR rebuild the sdk

import { getAiBackendKind } from './apiKeyStore.js';
import { getProxyJwt, setupJwtRotation } from './auth/proxyJwtFetcher.js';

const PROXY_BASE_URL = process.env.SEI_PROXY_URL ?? 'https://api.sei.gg';

export async function buildAnthropicConfig(): Promise<{
  apiKey?: string;
  cloudMode?: { baseURL: string; authToken: string };
}> {
  const kind = await getAiBackendKind();
  if (kind !== 'cloud-proxy') {
    // BYOK path — apiKey loaded from safeStorage as before
    const { loadAnthropicApiKey } = await import('./apiKeyStore.js');
    return { apiKey: await loadAnthropicApiKey() };
  }
  // Cloud-proxy path
  const jwt = await getProxyJwt();
  return {
    cloudMode: { baseURL: PROXY_BASE_URL, authToken: jwt },
  };
}

// At bot-session start, also wire up the rotation pump:
export async function startBotSessionWithRotation(messagePort: { postMessage: (m: any) => void }): Promise<() => void> {
  const kind = await getAiBackendKind();
  if (kind !== 'cloud-proxy') return () => undefined;
  return setupJwtRotation(messagePort);
}
```
