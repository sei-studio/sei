Section 4 of 8 — ledger_balance VIEW (REGULAR view, NOT materialized — open-question resolution #2 + research recommendation; document trade-off inline in the migration as a SQL comment):
```sql
-- NOTE: D-47 originally specified a materialized view. Per RESEARCH §Pattern 5 + Anti-Patterns,
-- we use a regular view to avoid per-insert refresh contention. Revisit if per-user row counts
-- exceed ~10K (i.e. when ledger_consumption.user_id sees >10K reservations).
create view public.ledger_balance as
select
  u.id as user_id,
  coalesce((select sum(credits_micro) from public.ledger_grants g where g.user_id = u.id), 0)::bigint
  - coalesce((select sum(c.micro) from public.ledger_consumption c
              where c.user_id = u.id and c.reservation_state in ('reserved','settled')), 0)::bigint
  as balance_micro
from auth.users u;
grant select on public.ledger_balance to authenticated, service_role;
```

Section 8 of 8 — RPCs (reserve_credits + settle_consumption, per RESEARCH §Pattern 5):
```sql
create or replace function public.reserve_credits(
  p_user_id uuid,
  p_reservation_micro bigint
) returns table(id uuid) language plpgsql security definer set search_path = public as $$
declare
  v_balance bigint;
  v_reservation_id uuid;
begin
  -- Lazy-insert the lock row (idempotent)
  insert into public.user_balance_lock (user_id) values (p_user_id)
    on conflict (user_id) do nothing;
  -- Acquire per-user FOR UPDATE row lock (Pattern 5 TOCTOU mitigation)
  perform 1 from public.user_balance_lock where user_id = p_user_id for update;
  -- Read balance from the regular view (live sum)
  select balance_micro into v_balance from public.ledger_balance where user_id = p_user_id;
  if v_balance is null or v_balance < p_reservation_micro then
    return; -- empty result set = insufficient
  end if;
  insert into public.ledger_consumption (user_id, micro, reservation_state)
  values (p_user_id, p_reservation_micro, 'reserved')
  returning ledger_consumption.id into v_reservation_id;
  return query select v_reservation_id;
end;
$$;
grant execute on function public.reserve_credits(uuid, bigint) to service_role;

create or replace function public.settle_consumption(
  p_reservation_id uuid,
  p_actual_micro bigint,
  p_anthropic_call_id text
) returns void language plpgsql security definer set search_path = public as $$
begin
  update public.ledger_consumption
  set
    -- LEAST guards against a buggy proxy sending actual > reservation. Refund delta is implicit
    -- in the ledger_balance view (lower micro on settled row → higher balance vs prior 'reserved').
    micro = LEAST(micro, p_actual_micro),
    reservation_state = 'settled',
    anthropic_call_id = p_anthropic_call_id
  where id = p_reservation_id and reservation_state = 'reserved';
end;
$$;
grant execute on function public.settle_consumption(uuid, bigint, text) to service_role;
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| auth.users → ledger writes | service_role-only (no RLS insert policies); proxy + Edge Functions are the trust anchor |
| user reading own balance | RLS select_own; cannot read other users' ledger rows |
| Two concurrent proxy requests for same user | user_balance_lock FOR UPDATE serializes balance reads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01-01 | Tampering | reserve_credits RPC | mitigate | FOR UPDATE on user_balance_lock serializes concurrent reservations per user; reading balance + INSERT happen inside the same lock-held transaction |
| T-13-01-02 | Tampering | ledger_grants.lemon_event_id | mitigate | UNIQUE constraint enforces webhook idempotency at the DB layer (re-INSERT raises 23505; lemon-webhook treats as success) |
| T-13-01-03 | Information Disclosure | ledger rows of other users | mitigate | RLS select_own scoped by auth.uid() prevents cross-user reads via PostgREST or supabase-js |
| T-13-01-04 | Elevation of Privilege | Hostile direct INSERT into ledger_grants via authenticated user | mitigate | No insert/update/delete RLS policies — only service_role (Edge Functions + proxy) can write. Defense in depth vs a compromised JWT |
| T-13-01-05 | Tampering | trial_claims after account deletion | mitigate | `on delete set null` retains the row so deleted accounts don't free the mc_username for re-claim (D-42a) |
| T-13-01-06 | DoS | rate_buckets unbounded growth | mitigate | pg_cron nightly delete of rows where window_start < now() - 25h |
| T-13-01-07 | Tampering | Proxy sends actual_micro > reservation in settle_consumption | accept | LEAST(micro, actual_micro) clamps. Proxy bug surfaces in logs (settle would lower balance below worst-case) — accept and monitor |
| T-13-01-08 | Spoofing | Direct call to reserve_credits as authenticated user | mitigate | Function granted only to service_role; not authenticated. authenticated callers receive permission_denied |
</threat_model>
