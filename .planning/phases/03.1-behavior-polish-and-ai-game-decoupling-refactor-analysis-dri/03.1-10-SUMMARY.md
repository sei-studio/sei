## Decisions Made

- **WR-01 RED test passed pre-fix; no fail-fast violation.** The buggy re-queue branch is unreachable under current scheduling because `scheduleProcess` gates on `!processing` and the trailing-drain `setImmediate(processNext)` only fires after the in-flight finally clears currentAction. The test suite is a regression baseline; removing the `processing = false` early-return is prophylactic and closes the latent stall against any future scheduling change. Documented in commit `ec3380c`.
- **`_setAbortController` over `Object.defineProperty`.** A `let abortController` closure variable plus a setter is simpler than redefining the property descriptor on every swap; the getter still returns the active reference and consumers re-read on each access.
- **WR-04 uses chained re-enqueue, not seed-addendum coupling.** P0 attack first, P1 chat second — the priority queue already orders these naturally. Avoids bespoke "owner also said: ..." textual coupling in the seed prompt.
- **`logger = console` default in composeSeedBlocks.** Keeps the function importable by test harnesses without breaking changes; the orchestrator call site plumbs the real logger.
