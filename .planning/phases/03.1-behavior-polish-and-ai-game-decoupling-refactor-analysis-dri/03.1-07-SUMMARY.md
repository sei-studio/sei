# Phase 03.1 Plan 07: Tone and Say Polish (Gap Closure) Summary

Polished say() output and loop-end emission so the bot sounds alive (commas + apostrophes survive), stops repeating itself byte-equal across loop_end ticks, and lets the model author its own iteration-cap wrap-up instead of using the static `capHitLine` string.

## Coverage

This plan addresses 3 of 17 Bucket A items from the Phase 03.1 verification report:

- **D-NEW-TONE-1** (#1) — punctuation: KEEP `,` `!` `?` `'`; STRIP `.` `—` `–` `"` `` ` ``
- **D-NEW-DM-1/2/3** (#2) — byte-identical loop_end re-emit chain
- **D-W-8 / D-NEW-TONE-2** (#5) — static cap-close line surfaced in wood-postfix

## Tasks Executed

### Task 1: Refine `postProcessSay` regex per D-NEW-TONE-1

**Commit:** `607607f` — `fix(03.1-07): postProcessSay keeps , ! ? ' ; : — strips only . — – " ` (D-NEW-TONE-1)`

Narrowed the strip regex from `[.,!?;:—–"\`]` to `[.—–"\`]`, matching the user's verbatim spec from `memory-postfix.txt` header item 5. Apostrophes already survived. New strip class strips ONLY periods, em-dash, en-dash, double quote, and backtick.

Updated `scripts/test-postProcessSay.mjs` to 14 cases including 4 new ones that prove preserved punctuation:
- `'oh, really? wow!'` → `'oh, really? wow!'`
- `'nope. done.'` → `'nope done'`
- `'en–dash and em—dash'` → `'en dash and em dash'`
- `'ok, im fine; thanks: see you'` → `'ok, im fine; thanks: see you'`

Code comment block records D-NEW-TONE-1 traceability and lists kept-vs-stripped chars explicitly.

**Verification:** `node scripts/test-postProcessSay.mjs` → `postProcessSay: all cases passed` (exit 0).

### Task 2: Add `shouldSuppressLoopEndSay` predicate (TDD, D-NEW-DM-1/2/3)

**RED commit:** `35cfb48` — `test(03.1-07): add failing test for shouldSuppressLoopEndSay (D-NEW-DM-1/2/3)`
**GREEN commit:** `f1386e2` — `feat(03.1-07): dedupeSay suppresses byte-equal loop_end say() within 2s (D-NEW-DM-1/2/3)`

**RED phase:** `scripts/test-dedupeSay.mjs` covers all 6 cases from the plan's `<behavior>` block plus a punctuation-only delta bonus case. Initially fails with `SyntaxError: does not provide an export named 'shouldSuppressLoopEndSay'`.

**GREEN phase:** Three changes made simultaneously (single commit, behavior is intertwined):
1. **`shouldSuppressLoopEndSay`** exported from `orchestrator.js` — returns `true` only when `triggerEvent === 'sei:loop_end'`, last self line is recent (<2000ms), AND normalized-equals candidate. Normalize = lowercase + non-alphanum→space + collapse-whitespace. Default window 2000ms, configurable via `windowMs` param.
2. **`convoMemory.recentChat.lastSelf()`** accessor added — returns `{ at, who, text }` of the most recent self-buffered line, or `null`.
3. **say() handler in `handleDispatch`** gated on the predicate. Suppressed lines are NOT chatted, NOT pushed to `recentChat.pushSelf`, and the tool_result content reads `"said (suppressed: byte-equal duplicate within 2s)"` so the LLM gets honest feedback. An info-level log `[sei/orch] dedupeSay suppressed loop_end duplicate (loop=…): …` makes the path observable in post-replay.

**Verification:** `node scripts/test-dedupeSay.mjs` → `dedupeSay: all cases passed` (exit 0). All 7 grep gates pass.

### Task 3: Cap-close uses model-authored line (D-W-8 / D-NEW-TONE-2)

**Commit:** `7cfd7f3` — `fix(03.1-07): cap-close uses model-authored wrap-up; capHitLine last-resort only`

Three targeted changes to `gracefulCapClose`:

1. **Strengthened seed prompt** (replaces "Wrap up with one short say."):
   > "You hit the iteration cap and have to stop. Write ONE short line — no more than 12 words, lowercase, no periods — that wraps up gracefully in your own voice. Acknowledge the limit briefly without being whiny. Examples in tone: 'alright, calling it for now', 'head's spinning, taking a beat', 'stopping here, will pick this up'. Output ONLY the line, no quotes, no explanation."
2. **Distinct logs** — info `cap-close: model-authored wrap-up` when model produced text, warn `cap-close: model returned empty text — falling back to capHitLine` when it did not. Post-replay debugging now tells us which path fired.
3. **`postProcessSay(modelText)`** runs the model's wrap-up through the same punctuation rules as a normal `say()` — closes the consistency gap (model emits raw text in its content block, not via the say tool, so `postProcessSay` was being skipped).
4. **Timeout bumped** to `Math.max(config.anthropic.timeout_ms, 8000)` for the cap-close call only — this is a single blocking call with no retry, so a slightly slow API response should not surface `capHitLine`.

`capHitLine` remains in `persona.js` as the last-resort fallback for genuine API failures (network errors, timeouts, empty response).

**Verification:** All 7 acceptance grep gates pass; `node -e "import('./src/brain/orchestrator.js').then(m=>console.log(typeof m.postProcessSay))"` prints `function`.

## Behavioral Impact (qualitative)

After this plan, in the next live replay we expect:

1. **Tone improvement** — say() output should sound natural with commas, exclamation marks, question marks. Bot can say `"oh, really? wow!"` instead of `"oh really wow"`.
2. **No more byte-identical re-emission** — the memory-postfix L21+L27 (13ms apart, identical bytes) and hunt+sand-postfix triple-emit pattern (`"plains is solid got some good loot too"` 3× in 3s across loops 16/17/18) should disappear. Post-replay logs will show `[sei/orch] dedupeSay suppressed loop_end duplicate` lines marking the suppressions.
3. **No more "okay, brain melting"** — when the iteration cap hits, the model authors a one-line wrap-up matching its own voice. Static `capHitLine` only surfaces if the API call fails or returns empty text (rare; logs will say `cap-close: model returned empty text — falling back to capHitLine`).

These claims are forward-looking; verification requires the next live replay session under similar conditions to wood-postfix / memory-postfix / hunt+sand-postfix.
