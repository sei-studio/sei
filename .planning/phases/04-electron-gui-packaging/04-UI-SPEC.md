# Phase 4 — UI Design Contract

> Visual and interaction contract for the Sei Electron launcher. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Source of truth:** `.planning/phases/04-electron-gui-packaging/design/project/` (HTML/JSX mockup) and `04-CONTEXT.md` decisions D-28..D-58 (verbatim from design iteration). This spec distills those into an implementation contract; on conflict the **mockup wins** for visuals, **CONTEXT wins** for behavior.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Electron desktop app — shadcn does not apply) |
| Preset | not applicable |
| Component library | hand-rolled primitives ported from `design/project/ui.jsx` (Button, TextField, IconRail, MacosWindow, PixelPortrait); no Radix / Headless UI |
| Icon library | inline SVG icons (BackIcon, ArrowIcon, PlusIcon, SparkleIcon, HomeIcon, SettingsIcon, SunIcon, MoonIcon) — copy verbatim from `ui.jsx`. No icon-font dependency. |
| Body font | **Noto Sans** — only weights 400 and 600 are loaded (`--sans`); see Typography for token scale |
| Pixel font | **Press Start 2P** (`--pixel`) — Sei wordmark, character-page title, LAN modal step numbers, "Other games" stub. **Never on body copy.** |
| Mono font | **JetBrains Mono** 400/500 (`--mono`) — eyebrows, status pills, API-key field, log viewer, persona-prompt body, settings rows, last-launched dates |
| Font delivery | **Bundled, not CDN.** Copy WOFF2s from Google Fonts to `src/renderer/public/fonts/` and `@font-face` from `src/renderer/src/styles/fonts.css`. Renderer must work offline (D-05 — no CDN React/Babel; same rule applies to fonts). |
| Border radius | **0 across every component** (D-28). All edges sharp. |
| Theme | light + dark; `prefers-color-scheme` on first run; user override persisted in `config.json.theme_mode: 'system' | 'light' | 'dark'` (D-33). Theme applied via `data-theme="light"|"dark"` attribute on `<html>`. |

## Typography

Strict token scale: **exactly 4 sans sizes, exactly 2 sans weights.** Anything outside this set is a component-specific override (justified per row below) — not a token.

### Sans token scale (Noto Sans)

| Token | Size | Weight | Line height | Letter spacing | Where |
|-------|------|--------|-------------|----------------|-------|
| `display` | 30px (32px on HomeScreen "Characters") | 600 | 1.2 | -0.4 | Onboarding question title, Home h1, Settings h1, ComingSoon h1 |
| `body` | 16px | 400 | 1.5 | 0 | TextField input, hint paragraphs, body copy |
| `label` | 14px | 600 | 1.2 | -0.1 | Button labels, Settings row label, theme toggle row, AddCard label, SummonToast title |
| `caption` | 11px | 400 | 1.3 | 0 | Card status chip, "Last:" stamp, never-summoned text, captions |

**Weights:** only `400` (regular) and `600` (bold) are loaded. The `display` token paired with `weight: 600` covers all heading emphasis. Where the prior draft used weight `500`, those usages are folded into `400` (with letter-spacing -0.1 if visual weight needs to nudge up) or promoted to `600` — see "Sans component overrides" below for per-site assignment.

### Accent reserved-for list (D-29 enforcement)

Accent (`--accent` / `--accent-strong` / `--accent-soft`) is allowed **only** on these elements:

1. Primary "Begin" / "Finish" / "Create" / "Summon" buttons (Button kind=`accent`)
2. Active step dot in onboarding / add-character (StepDots current index)
3. Selected provider tile border + 30% wash background
4. Sidebar active-view 3px left bar
5. AddCard hover state (border, label color, fill icon)
6. Persona-prompt expanded card left-border (2px) + Show/Hide toggle text
7. Sei wordmark recolor (LoadingScreen, onboarding "Welcome to [Sei]." mark)
8. LAN modal step numbers (01–04)
9. ComingSoon "Other games" pixel header
10. TextField underline on `:focus`
11. Hover color on inactive Sidebar RailButton (icon stroke)
12. AddCharacter step-2 ("Create") accent CTA, onboarding step-0 "Begin" CTA, onboarding step-4 "Finish" CTA
13. ComingSoon stub eyebrow

Accent is **forbidden** on: body text, links inside body copy, hint paragraphs, breadcrumb chrome, settings row chrome, log lines (color-tagging uses semantic colors not accent), Cancel / Close / Back buttons (those are `quiet` or `ghost`).

### Destructive use list

`--red` is allowed **only** on:
1. Delete button label color in CharacterPage (visible only when `id !== 'sui'`).
2. "Not connected" pill dot (HomeScreen + LAN modal header).
3. Error-state dot in model row (replaces green when bot reports error).
4. (Implicit) form validation hint copy if a future error surface is added.

## Component Inventory

Each component below MUST be ported as a real React component under `src/renderer/src/components/`. Inline-style mockup is for visual reference; production should use CSS modules or a styled-components/Tailwind layer (Defaults section picks one).

### Button

Source: `ui.jsx` Button.

Props:
```ts
{ children, onClick, kind?: 'primary' | 'accent' | 'ghost' | 'quiet',
  size?: 'sm' | 'md' | 'lg', icon?: ReactNode, disabled?: boolean,
  fullWidth?: boolean, type?: 'button' | 'submit',
  'aria-label'?: string }
```

Variants × size matrix:

| size | h | px | fs | gap |
|------|----|----|----|-----|
| sm | 32 | 12 | 13 | 6 |
| md | 38 | 16 | 14 | 8 |
| lg | 46 | 22 | 15 | 10 |

| kind | bg | color | border | hover bg | shadow |
|------|-----|-------|--------|----------|--------|
| primary | `--text` | `--window` | transparent | `--text` (no change) | `0 1px 2px / hover 0 4px 14px rgba(0,0,0,.15)` |
| accent | `--accent` | `--accent-text` (`#2A1B12` light / `#0F1218` dark) | transparent | `--accent-strong` | hover `0 6px 20px var(--accent-soft)` |
| ghost | transparent | `--text` | `--border` | `--surface` | none |
| quiet | transparent | `--text-2` | transparent | `--surface` | none |

States:
- **default** — as above
- **hover** — translateY(-0.5px), bg/shadow swap
- **focus-visible** — 1.5px outline `var(--accent)` offset 2px; sharp corners (no rounded outline since radius is 0); never replace native `:focus-visible`. Apply via `outline` on the `<button>`.
- **disabled** — opacity 0.5, cursor not-allowed, no hover transform; on Provider tile (D-26) `aria-disabled="true"` + "Coming soon" chip on the disabled tiles (OpenAI / Google / Local).
- **active** — no extra style needed; press feel comes from translateY snap-back (transition 80ms transform).

### PixelPortrait

Source: `ui.jsx` PixelPortrait.

Props: `{ seed: string, palette: string[], size?: number, style?: CSSProperties }`.

Algorithm (D-14, port verbatim — covered by determinism contract under "Interaction Contracts"):
1. `hashSeed(seed)` — FNV-1a 32-bit from string char codes.
2. xorshift PRNG: `s = (Math.imul(s ^ (s>>>15), 2246822507) ^ Math.imul(s ^ (s>>>13), 3266489909)) >>> 0`; output `s/4294967296`.
3. 12 × 12 grid; left half (cols 0..5) generated, right half mirrored.
4. Top 2 rows = sky (background `palette[0]` → `palette[1]` linear gradient).
5. Col 0 and last row force background.
6. Rows 7..11 (`bodyStart=7`) — 75% non-background fill, palette index `2 + floor(rng * (palLen-3))`.
7. Rows 2..6 (head) — 82% fill, palette index `1 + floor(rng * (palLen-2))`.
8. Eye pixels at `(row=4, col=3)` and `(row=4, col=8)` forced to `#0E0E0E`.
9. Palette: 6 colors, deterministically picked from a fixed list keyed by `hashSeed(name)` — port the list from `app.jsx` (light theme + dark theme variants).

Override: if `<userData>/characters/<id>.png` exists, render that image instead of the procedural sprite, scaling via `image-rendering: pixelated`. Missing file → fall back to procedural (D-14).

## Interaction Contracts

### Summon flow

1. User presses Summon (CharacterCard hover overlay button OR CharacterPage primary CTA).
2. Renderer reads cached `lanState` from `onLan` subscription:
   - **`connected`** → fire SummonToast immediately, IPC `bot:summon(characterId)` to main with cached `{port}`. Status row flips to "Connecting…" (yellow dot intermediate).
   - **`not_connected` / `unavailable`** → open LAN modal in Searching mode (D-24). Pending summon held in renderer state. When `onLan` flips to `connected`: dismiss modal, fire toast, IPC summon. ESC or Cancel summon: drop pending summon, no IPC.
3. Main spawns `utilityProcess.fork('src/bot/index.js')` with merged config (D-18) over MessagePort.
4. Bot lifecycle events drive UI:
   - `connected` → model row shows "Online" with uptime ticker; Logs tab enables.
   - `disconnected` → model row shows "Disconnected · Reconnecting…" with `--muted` dot.
   - `error` → model row shows red dot + plain-English message + "Try again" link.
   - `summon-stopped` → model row reverts to "Ready"; Logs tab disables; lastLaunched + playtime update.
5. **Summon-while-active** (D-16): if a different character is summoned while one is running, the supervisor stops the current bot first (10s `bot.quit()` budget), then starts the new one. Renderer shows two SummonToasts in sequence ("Stopping {old}…" then "Summoning {new}…").

### LAN auto-resume

D-24 / D-56. Modal subscribed to `onLan`. State machine:

```
            ┌─────────────────────┐
press Summon│ summonPending=true  │
  + lan!=ok │ modal=Searching     │──── onLan('connected') ──► dismiss + IPC bot:summon(id, port)
            └─────────────────────┘
                    │ Cancel summon / ESC
                    ▼
            summonPending=false
            modal closed
```

Modal also openable from the LAN pill in HomeScreen (info-mode, no Searching banner, no Cancel summon button).

### Theme toggle

- Sidebar theme button cycles `light ↔ dark`. (No "system" cycle in the toggle button — system mode only set via initial detection or programmatic Settings input.)
- Persists `theme_mode` to `<userData>/config.json` immediately.
- Applies via `document.documentElement.setAttribute('data-theme', mode)`.
- `prefers-color-scheme` listener: only respected when `theme_mode === 'system'` (which is the first-run default).

### Character delete-gating

- Delete button **never rendered** when `character.id === 'sui'` (D-49).
- For other characters: clicking Delete opens a sharp-cornered confirm modal (lift LanModal styling) with copy:
  - Title: "Delete {name}?"
  - Body: "This permanently removes their persona, description, and saved memory. You can't undo this."
  - Buttons: "Cancel" (kind=quiet) / "Delete {name}" (kind=primary, color override `--red` background, `--window` text).
- Confirmation deletes:
  - `<userData>/characters/<id>.json`
  - `<userData>/characters/<id>.png` if exists
  - `<userData>/memory/<id>/` directory recursively
  - `index.json` entry removed
- Then navigates back to HomeScreen with a transient toast "Deleted {name}".

### Re-onboarding

- Triggered from Settings → "Start over". Full Onboarding flow in `isReonboard=true` mode.
- Pre-fills with current `config.json` values (mcUsername / preferredName / provider).
- API-key field starts empty (force re-entry; do not pre-fill safeStorage value).
- On Finish: write new values, replace API key in safeStorage, navigate to Home.

### PixelPortrait determinism

Same `seed = id + name` always produces same sprite within a theme. Palette (6 colors) is selected from a fixed list of 8–10 pre-curated palettes per theme (port from `app.jsx` — verify list during planning). Theme switch preserves the sprite layout but swaps palette.

Override: if `<userData>/characters/<id>.png` is present, render image instead — preserves user-uploaded portrait. Loss of file → silent fallback to procedural.

## Implementation Notes (carry-forward)

These are non-binding hints for the planner / executor — they're already implied by CONTEXT but worth surfacing so they don't get lost between docs:
