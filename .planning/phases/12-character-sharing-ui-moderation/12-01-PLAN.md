**Section 4 of 4** — append two trigger functions + their triggers.

```sql
-- pg_net required for Database Webhook (Pattern 1 in 12-RESEARCH.md).
create extension if not exists pg_net with schema extensions;

-- Auto-hide: 3 distinct reporters in 24h → characters.shared = false.
-- AFTER INSERT (not BEFORE) so the count includes the new row.
create or replace function public.tg_reports_auto_hide()
returns trigger
language plpgsql
security definer
set search_path = public
as $$
declare
  distinct_reporters int;
begin
  select count(distinct reporter_id) into distinct_reporters
    from public.reports
   where character_id = new.character_id
     and created_at > now() - interval '24 hours';

  if distinct_reporters >= 3 then
    update public.characters
       set shared = false
     where id = new.character_id
       and shared = true;   -- guard avoids duplicate auto-hides
  end if;
  return new;
end;
$$;

create trigger reports_auto_hide_trigger
  after insert on public.reports
  for each row execute function public.tg_reports_auto_hide();

-- Database Webhook: report INSERT → notify-report Edge Function via pg_net.
-- Per Pitfall 2: pg_notify does NOT cross to Edge Functions. Use pg_net.http_post.
-- Per Assumption A8: edge_url + service_role_key MUST be set out-of-band via
--   ALTER DATABASE postgres SET app.settings.edge_url = 'https://<ref>.supabase.co';
--   ALTER DATABASE postgres SET app.settings.service_role_key = '<key>';
-- Document this in 12-NN-SUMMARY.md so operator runbook captures it.
create or replace function public.tg_notify_report_inserted()
returns trigger
language plpgsql
security definer
set search_path = public, extensions
as $$
declare
  edge_url    text;
  service_key text;
begin
  edge_url    := current_setting('app.settings.edge_url',           true)
                 || '/functions/v1/notify-report';
  service_key := current_setting('app.settings.service_role_key',  true);

  if edge_url is null or service_key is null then
    -- Don't fail the INSERT; just no-op. Operator must set the settings
    -- before notifications start flowing.
    raise notice 'notify-report webhook skipped: app.settings.* not configured';
    return new;
  end if;

  perform net.http_post(
    url     := edge_url,
    headers := jsonb_build_object(
      'Content-Type', 'application/json',
      'Authorization', 'Bearer ' || service_key
    ),
    body    := jsonb_build_object('report_id', new.id, 'character_id', new.character_id)
  );
  return new;
end;
$$;

create trigger reports_after_insert_webhook
  after insert on public.reports
  for each row execute function public.tg_notify_report_inserted();
```

**Trigger order note:** both triggers fire AFTER INSERT FOR EACH ROW. Postgres fires them in **alphabetical order by trigger name**: `reports_after_insert_webhook` fires AFTER `reports_auto_hide_trigger`. This is correct — the webhook payload includes the up-to-date `characters.shared` value (auto-hide already applied) so notify-report can detect the auto-hide case via a secondary SELECT.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anon JWT → Postgres RPC | Untrusted query string crosses here (search_query) |
| Database trigger → Edge Function HTTP | Internal RPC over network — Authorization header carries service_role_key |
| Service-role caller → reports INSERT | Only path that can write reports rows |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01-01 | Tampering | reports table — fake reporter_id | mitigate | No insert-own RLS policy; submit-report Edge Function reads reporter_id from auth.getUser() never from request body (Pitfall 4 + Plan 12-05) |
| T-12-01-02 | Spoofing | Database Webhook — anyone could call notify-report directly | mitigate | Edge Function checks `Authorization: Bearer <service_role>` matches `Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')` (Plan 12-06) |
| T-12-01-03 | Information Disclosure | search_public_characters leaking shared=false rows | mitigate | security invoker (DEFAULT) honors caller RLS; explicit anti-test: anon query MUST return 0 rows where shared=false |
| T-12-01-04 | Denial of Service | ILIKE wildcard search with no row cap | mitigate | `least(page_limit, 50)` hard cap in RPC (Pattern 3) |
| T-12-01-05 | Repudiation | Auto-hide UPDATE leaves no audit trail | accept | v1.0: characters.updated_at trigger captures timestamp; admin SELECT on reports table reconstructs history. Audit log out of scope. |
| T-12-01-06 | Elevation | tg_reports_auto_hide is security definer — could be exploited to UPDATE arbitrary chars | mitigate | function body only UPDATEs WHERE id = NEW.character_id (validated by FK on reports.character_id); search_path pinned to public |
| T-12-01-07 | Tampering | Trigger ordering race (webhook fires before auto-hide) | mitigate | Trigger names sorted alphabetically: auto_hide < webhook → auto-hide runs first; webhook payload sees post-auto-hide state. Document in operator runbook. |
